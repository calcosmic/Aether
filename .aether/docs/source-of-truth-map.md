# Aether Source-of-Truth Map

Updated: 2026-05-04

This document defines which files are authoritative, which files are installed
globally, and which repo-local files belong to each colony.

For dummies: edit the real source file once in the Aether repo, publish it to
the hub, and keep target repos focused on their own state.

## Authority Order

1. **Executable runtime**
   - `cmd/`
   - `pkg/`
   - Why: this is the Go implementation the `aether` binary runs.

2. **Canonical platform sources**
   - `.claude/agents/ant/*.md`
   - `.claude/commands/ant/*.md`
   - `.opencode/agents/*.md`
   - `.opencode/commands/ant/*.md`
   - `.codex/agents/*.toml`
   - Why: these are the source files installed globally for each platform.

3. **Aether companion sources**
   - `.aether/commands/*.yaml`
   - `.aether/skills/**/SKILL.md`
   - `.aether/templates/`
   - `.aether/docs/`
   - `.aether/utils/`
   - `exchange/` modules under `.aether/`
   - `.aether/references/`
   - `workers.md` under `.aether/`
   - Why: these publish into the global hub and are no longer copied into every
     target repo.

4. **Repo-local state**
   - `.aether/data/`
   - `.aether/dreams/`
   - `.aether/oracle/`
   - `.aether/locks/`
   - `.aether/QUEEN.md`
   - `.aether/skills/` for custom repo skills only
   - Why: these are generated by work in one repo and must not be shared as
     system assets.

## Ownership Map

| Area | Source of truth | Installed or consumed from |
|---|---|---|
| Go runtime | `cmd/`, `pkg/` | `aether` binary |
| Claude agents | `.claude/agents/ant/*.md` | global Claude agent directory |
| Claude commands | `.claude/commands/ant/*.md` | global Claude command directory |
| OpenCode agents | `.opencode/agents/*.md` | global OpenCode agent directory |
| OpenCode commands | `.opencode/commands/ant/*.md` | global OpenCode command directory |
| Codex agents | `.codex/agents/aether-*.toml` | global Codex agent directory and hub `system/codex/` |
| Shipped skills | `.aether/skills/**/SKILL.md` | hub skill directories and global Codex skill install |
| Slash-command wrapper specs | `.aether/commands/*.yaml` | generated Claude/OpenCode wrappers |
| Templates/docs/utils/exchange/references/workers | Aether repo companion source paths | hub `system/` only |
| Colony state | repo `.aether/data/` | current repo only |
| Repo wisdom | repo `.aether/QUEEN.md` | current repo only |
| Custom repo skills | repo `.aether/skills/` | current repo only |

## Maintenance Rules

1. Change runtime behavior in `cmd/` or `pkg/` first.
2. Change platform agents and command wrappers in their canonical platform source
   directories.
3. Change shipped skills only under `.aether/skills/`.
4. Publish from the Aether repo with `aether publish` or
   `aether install --package-dir "$PWD"`.
5. In target repos, run `aether update` to refresh local scaffolding and prune
   old generated repo-local asset copies.
6. Never treat repo `.aether/data/`, `.aether/dreams/`, `.aether/oracle/`,
   `.aether/locks/`, `.aether/QUEEN.md`, or custom repo skills as shipped
   system assets.
7. Do not recreate retired packaging mirror directories under `.aether/`; edit
   canonical platform sources and command YAML instead.

## Wrapper-Runtime Boundary

The Go runtime owns state mutations, verification, persistence, file locking,
phase transitions, and visual rendering. Wrapper markdown may add platform
presentation, but it must not duplicate runtime state logic.

Codex does not use wrapper markdown. Codex UX comes from the `aether` CLI and
Codex TOML agent definitions.
