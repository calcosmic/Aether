# Requirements: v3.0 Wisdom & Pheromone Evolution

**Defined:** 2026-02-20
**Core Value:** Transform pheromones from manual signals into an intelligent, self-improving wisdom system that learns from every colony

## v3.0 Requirements

### Pheromone Evolution

- [x] **PHER-EVOL-01**: Pheromones automatically injected at key workflow points (build reads pheromones, primes workers)
- [x] **PHER-EVOL-02**: End-of-phase pheromone proposals displayed for user approval (continue extracts learnings, proposes)
- [x] **PHER-EVOL-03**: Tick-to-approve UX for proposed pheromones (user selects which to promote)
- [x] **PHER-EVOL-04**: Pheromone history tracking in colony state (signals persist with timestamps)

### QUEEN.md System

- [x] **QUEEN-01**: QUEEN.md file structure with 5 wisdom categories (Philosophies, Patterns, Redirects, Stack Wisdom, Decrees)
- [x] **QUEEN-02**: queen-init command creates QUEEN.md from template (called by init.md)
- [x] **QUEEN-03**: queen-read command returns wisdom as JSON for worker priming (called by build.md)
- [x] **QUEEN-04**: queen-promote command with type validation (enforces thresholds)
- [x] **QUEEN-05**: Metadata block with version, stats, thresholds (in HTML comment)

### Command Integration

- [x] **INT-01**: init.md calls queen-init after bootstrap (QUEEN.md created)
- [x] **INT-02**: build.md calls queen-read before spawning workers (wisdom injected)
- [x] **INT-03**: continue.md promotes validated learnings to QUEEN.md (with approval)
- [x] **INT-04**: seal.md promotes final colony wisdom (before sealing)
- [x] **INT-05**: entomb.md promotes wisdom before archiving (before entomb)

### Metadata & Tracking

- [x] **META-01**: Promotion thresholds enforced per wisdom type (philosophy:5, pattern:3, redirect:2, stack:1, decree:0)
- [x] **META-02**: Evolution log tracks wisdom changes over time (in QUEEN.md)
- [x] **META-03**: Stats block tracks counts per category (total_philosophies, etc.)
- [x] **META-04**: colonies_contributed tracks wisdom origins (which colonies added what)

### Observation Tracking

- [x] **OBS-01**: learning-observe function records observations to learning-observations.json
- [x] **OBS-02**: learning-check-promotion function returns proposals meeting thresholds
- [x] **OBS-03**: Observation counts accumulate across colonies (not just per-colony)
- [x] **OBS-04**: Content hashing prevents duplicate observation entries

### Unified Priming

- [x] **PRIME-01**: colony-prime function combines wisdom + signals + instincts
- [x] **PRIME-02**: build.md uses colony-prime for unified worker context
- [x] **PRIME-03**: Workers receive structured colony context (not multiple overlapping calls)

## v3.x Requirements (Deferred)

### Enhanced UX

- **UX-01**: Interactive tick-to-approve with multi-select
- **UX-02**: Wisdom search and filter in status command
- **UX-03**: Wisdom diff between colonies

### Cross-Colony Sync

- **SYNC-01**: Export wisdom to external file
- **SYNC-02**: Import wisdom from another project
- **SYNC-03**: Merge wisdom from multiple colonies

## Out of Scope

| Feature | Reason |
|---------|--------|
| Auto-promotion without approval | Causes wisdom drift from user intent |
| Global wisdom across all projects | Creates cross-project conflicts |
| Wisdom versioning/rollback | Premature complexity |
| AI-generated pheromones | Signal noise, wrong priorities |
| Real-time wisdom sync | Over-engineering for current needs |
| queen-wisdom.json redundancy | Remove in favor of QUEEN.md only |
