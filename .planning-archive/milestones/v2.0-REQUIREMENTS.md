## Agent Power Standards

Every agent must be genuinely powerful — not a mechanical port, but a best-in-class subagent comparable to superpowers/everything-claude-code quality.

- [ ] **PWR-01**: Every agent has a detailed execution flow (XML `<execution_flow>` with numbered steps for complex agents, clear headers for simple ones)
- [ ] **PWR-02**: Every agent has critical rules that prevent common failure modes (not generic advice, agent-specific discipline)
- [ ] **PWR-03**: Every agent has structured return format — what it outputs and how the calling command consumes it
- [ ] **PWR-04**: Every agent has success criteria — self-verification checklist before returning
- [ ] **PWR-05**: Every agent has failure modes — what to do when stuck, when to escalate, explicit retry limits
- [ ] **PWR-06**: Routing-effective descriptions ("Use this agent for X. Spawned by Y." — not vague role labels)
- [ ] **PWR-07**: Explicit tools field on every agent — no implicit inheritance, read-only agents verified
- [ ] **PWR-08**: All OpenCode-specific patterns removed — no spawn calls, no activity-log requirements, graceful degradation when colony not initialized
