# Requirements: Aether v5.0 Agent Integration

**Defined:** 2026-02-21
**Core Value:** Integrate specialist agents into existing commands to improve verification quality, security coverage, and efficiency

---

## The Opportunity

13 agent definitions exist in `.opencode/agents/` but are never spawned. The llm-architect analysis identified 8 worth integrating:

- **P0 (Critical):** Gatekeeper + Auditor — Security and quality gates
- **P1 (Important):** Probe + Measurer — Coverage and performance
- **P2 (Valuable):** Chronicler + Ambassador — Documentation and API integration
- **P3 (Nice-to-have):** Sage + Weaver — Analytics and proactive refactoring

**Result:** Existing commands gain specialist capabilities without new command surface.

---

## v5.0 Requirements

### P0 - Security Gates (Phase 38)

- [x] **SEC-01**: Gatekeeper spawns in `/ant:continue` Phase 5.5 when package manifest exists
- [ ] **SEC-02**: Gatekeeper performs CVE scanning, license compliance, supply chain security audit
- [ ] **SEC-03**: Gatekeeper blocks on critical CVEs, warns on high CVEs
- [ ] **SEC-04**: Auditor spawns in `/ant:continue` Step 1.8.5 when UI or API changes detected
- [ ] **SEC-05**: Auditor performs multi-lens review (security, performance, quality, maintainability)
- [x] **SEC-06**: Auditor gate fails if overall_score < 60 or critical findings > 0

### P1 - Quality Coverage (Phase 39)

- [ ] **COV-01**: Probe spawns in `/ant:continue` Phase 4.5 when coverage < 80% after tests pass
- [ ] **COV-02**: Probe generates tests for uncovered code paths
- [ ] **COV-03**: Probe discovers edge cases through mutation testing
- [ ] **COV-04**: Probe is non-blocking — continues even if can't improve coverage
- [ ] **COV-05**: Measurer spawns in `/ant:build` Step 5.5 for performance-sensitive phases
- [ ] **COV-06**: Measurer establishes performance baselines for new code
- [ ] **COV-07**: Measurer identifies bottlenecks and provides recommendations

### P2 - Lifecycle Enhancement (Phase 40)

- [ ] **LIF-01**: Chronicler spawns in `/ant:seal` Step 5.5 before ceremony display
- [ ] **LIF-02**: Chronicler surveys documentation coverage (API docs, READMEs, guides)
- [ ] **LIF-03**: Chronicler reports gaps but doesn't block seal
- [ ] **LIF-04**: Ambassador replaces Builder when task involves external API/SDK/OAuth
- [ ] **LIF-05**: Ambassador handles rate limiting, circuit breakers, retry patterns
- [ ] **LIF-06**: Ambassador returns integration plan for Builder to execute

### P3 - Analytics & Improvement (Phase 41)

- [ ] **ANA-01**: Sage spawns in `/ant:seal` Step 3.5 when colony has 3+ completed phases
- [ ] **ANA-02**: Sage analyzes velocity trends, bug density, review turnaround
- [ ] **ANA-03**: Sage provides data-driven insights to wisdom promotion process
- [ ] **ANA-04**: Weaver spawns in `/ant:continue` Step 1.7 when complexity exceeds thresholds
- [ ] **ANA-05**: Weaver refactors code to improve maintainability
- [ ] **ANA-06**: Weaver runs tests before and after — reverts if break
