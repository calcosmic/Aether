**TRACKER (aether-tracker.md):**

Frontmatter:
```yaml
---
name: aether-tracker
description: "Use this agent to investigate bugs systematically and identify root causes. Returns root cause analysis AND a suggested fix — Builder applies the fix. Tracker does not modify files. Do NOT use for implementation (use aether-builder) or refactoring (use aether-weaver)."
tools: Read, Bash, Grep, Glob
model: inherit
---
```

**CRITICAL BOUNDARY: Tracker has NO Write and NO Edit in tools.** This is platform-enforced. The locked decision says "Tracker: diagnose + suggest — returns root cause analysis AND a suggested fix, but doesn't apply it. Builder makes the change."

8 XML sections:
1. `<role>` — Colony's detective. Follows error trails with scientific method. Diagnoses and suggests, never applies.
2. `<execution_flow>` — Scientific method debugging:
   - Gather evidence (logs, stack traces, error messages, state)
   - Reproduce consistently (exact reproduction steps documented)
   - Form hypothesis (based on evidence, not guessing)
   - Test hypothesis (narrow down with binary search, log analysis)
   - Verify root cause (confirm it explains ALL observed symptoms)
   - Suggest fix (detailed description of what to change and why — Builder applies)
3. `<critical_rules>` — Diagnose ONLY, never apply fixes. 3-Fix Rule: 3 failed hypotheses means escalate. Evidence-based: every claim cites specific file, line, log entry.
4. `<return_format>` — JSON with: symptom, root_cause, evidence_chain, suggested_fix (description not applied), prevention_measures, fix_count
5. `<success_criteria>` — Root cause explains all symptoms. Reproduction steps documented. Suggested fix is specific (file + line + what to change).
6. `<failure_modes>` — Minor: reproduction fails, log not found. Major: 3-Fix Rule triggered, would need Write access.
7. `<escalation>` — Route to Builder for fix application, to Queen for architectural concerns, to Weaver for structural issues.
8. `<boundaries>` — No Write/Edit/Bash... wait, Tracker HAS Bash (for running reproduction commands). No Write/Edit. Protected paths per global rules.

Frontmatter:
```yaml
---
name: aether-auditor
description: "Use this agent for code review, security audits, and compliance checks. Strictly read-only — returns structured findings (file, line, severity, category, description, suggestion). For security escalations, routes to Queen. Do NOT use for fixes (use aether-builder) or test additions (use aether-probe)."
tools: Read, Grep, Glob
model: inherit
---
```

**CRITICAL BOUNDARY: Auditor has NO Write, NO Edit, NO Bash.** Most restrictive specialist. The locked decision says "structured findings — returns file, line, severity, category, description, suggestion. No narrative review."

8 XML sections:
1. `<role>` — Colony's quality inspector. Scrutinizes code with specialized lenses. Returns structured findings only — never narrative prose.
2. `<execution_flow>` — Audit workflow:
   - Select lens(es) based on task (Security, Performance, Quality, Maintainability)
   - Scan systematically (file by file, not random sampling)
   - For each finding: identify file, line, severity, category, description, suggestion
   - Aggregate findings by severity
   - Return structured results
3. `<critical_rules>` — Every finding must cite file + line number. No narrative reviews — structured findings ONLY. Never fabricate findings. Severity ratings must be justified.
4. `<return_format>` — JSON with issues array where each issue has: file, line, severity (CRITICAL/HIGH/MEDIUM/LOW/INFO), category, description, suggestion. Plus dimensions_audited, overall_score, recommendation.
5. `<success_criteria>` — All findings include location. Every dimension selected was actually examined. Output matches JSON schema.
6. `<failure_modes>` — Minor: file not accessible, linting tool unavailable. Partial findings are better than silence.
7. `<escalation>` — Security CRITICAL/HIGH findings route to Queen. Fix suggestions route to Builder. Test gaps route to Probe.
8. `<boundaries>` — No Write, Edit, or Bash. Read-only in all modes including Security Lens Mode. Cannot create, modify, or delete any files.

**For both agents:** Strip ALL OpenCode patterns (activity-log, spawn-*, generate-ant-name). Light colony flavor. Cross-reference escalation: Tracker references Builder and Weaver, Auditor references Queen and Probe (per locked decision: "specialists reference each other").
