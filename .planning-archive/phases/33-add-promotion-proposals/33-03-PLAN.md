<task type="auto">
  <name>Task 3: Add QUEEN-04, META-02, META-04 - queen-promote type validation and metadata</name>
  <files>.aether/aether-utils.sh</files>
  <action>
    Add type validation, threshold enforcement, and metadata tracking to the queen-promote function.

    The function should:
    1. Validate wisdom_type is one of: philosophy, pattern, redirect, stack, decree
    2. Check if content meets threshold for that type:
       - philosophy: 5 observations required
       - pattern: 3 observations required
       - redirect: 2 observations required
       - stack: 1 observation required
       - decree: 0 observations required (user mandates don't need thresholds)
    3. If threshold not met, return error with explanation
    4. If threshold met (or decree), proceed with promotion

    5. Update QUEEN.md metadata (JSON inside HTML comment) to track:
       a) META-02 - evolution_log: Append entry with timestamp, action="promote", wisdom_type, content_hash
       b) META-04 - colonies_contributed: Update mapping of which colonies contributed to which wisdom

    For HTML comment JSON manipulation in QUEEN.md:

    Step A: Extract existing metadata from HTML comment
    ```bash
    # Read first 20 lines, extract content between <!-- and -->
    meta_line=$(head -20 "$queen_file" | grep -E '^<!--\s*\{')
    if [[ -n "$meta_line" ]]; then
      # Extract JSON between <!-- and -->
      meta_json=$(echo "$meta_line" | sed 's/^<!--\s*//' | sed 's/\s*-->$//')
    else
      meta_json='{"version":"1.0","evolution_log":[],"colonies_contributed":{}}'
    fi
    ```

    Step B: Update evolution_log (META-02)
    ```bash
    new_entry=$(jq -n \
      --arg ts "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
      --arg action "promote" \
      --arg type "$wisdom_type" \
      --arg hash "$content_hash" \
      '{timestamp: $ts, action: $action, wisdom_type: $type, content_hash: $hash}')

    updated_meta=$(echo "$meta_json" | jq --argjson entry "$new_entry" \
      '.evolution_log += [$entry]')
    ```

    Step C: Update colonies_contributed (META-04)
    ```bash
    # colonies array comes from learning-observations.json
    updated_meta=$(echo "$updated_meta" | jq \
      --arg hash "$content_hash" \
      --argjson colonies "$(printf '%s\n' "${colonies[@]}" | jq -R . | jq -s .)" \
      '.colonies_contributed[$hash] = $colonies')
    ```

    Step D: Write updated metadata back to HTML comment
    ```bash
    # Replace first HTML comment line with updated JSON
    new_comment="<!-- $(echo "$updated_meta" | jq -c .) -->"
    awk -v new="$new_comment" 'NR==1 && /^<!--/ {print new; next} {print}' "$queen_file" > "${queen_file}.tmp"
    mv "${queen_file}.tmp" "$queen_file"
    ```

    Reference the existing QUEEN.md structure at .aether/docs/QUEEN.md for the format.
  </action>
  <verify>grep -n "queen-promote" .aether/aether-utils.sh | head -5</verify>
  <done>queen-promote validates type, enforces thresholds, and updates metadata (evolution_log and colonies_contributed)</done>
