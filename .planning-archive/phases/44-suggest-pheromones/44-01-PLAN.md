<task type="auto">
  <name>Task 1: Add suggest-analyze command to aether-utils.sh</name>
  <files>.aether/aether-utils.sh</files>
  <action>
    Add a new `suggest-analyze` command to aether-utils.sh that analyzes the codebase and returns pheromone suggestions.

    The command should:
    1. Accept optional `--source-dir` parameter (default: auto-detect src/, lib/, or current directory)
    2. Accept optional `--max-suggestions` parameter (default: 5)
    3. Read existing pheromones from pheromones.json to avoid duplicates
    4. Read session.json to check `suggested_pheromones` array for session deduplication
    5. Analyze files with these heuristics:
       - Large files: `wc -l > 300` → FOCUS "Large file: consider refactoring"
       - TODO/FIXME: `grep -rn "TODO\|FIXME\|XXX"` → FEEDBACK "N pending TODOs in modified files"
       - Debug artifacts: `grep -rn "console.log\|debugger"` → REDIRECT "Remove debug artifacts before commit"
       - Type gaps: `grep -rn ": any\|: unknown"` → FEEDBACK "Type safety gaps detected"
       - Complexity: `grep -c "^function\|^def\|^const.*=" > 20` → FOCUS "Complex module: test carefully"
       - Test gaps: Files without corresponding `.test.` files → FOCUS "Add tests for uncovered modules"
    6. Filter out suggestions already in pheromones.json (active signals)
    7. Filter out suggestions already in session.json suggested_pheromones
    8. Score and prioritize: REDIRECT (high) > FOCUS (normal) > FEEDBACK (low)
    9. Return JSON: `{ "suggestions": [...], "analyzed_files": N, "patterns_found": N }`

    Each suggestion object should have:
    - `type`: "FOCUS", "REDIRECT", or "FEEDBACK"
    - `content`: Human-readable suggestion text
    - `file`: File path (if file-specific)
    - `reason`: Brief explanation of why this was suggested
    - `hash`: sha256 of file+type+content for deduplication
    - `priority`: 1-10 score (higher = more important)

    Exclusions (never analyze):
    - node_modules/
    - .aether/
    - dist/
    - build/
    - *.min.js
    - .git/
    - coverage/

    Use existing patterns from surveyor-pathogens agent for grep commands.
  </action>
  <verify>
    bash .aether/aether-utils.sh suggest-analyze --dry-run returns valid JSON with suggestions array
  </verify>
  <done>
    suggest-analyze command exists, returns JSON with suggestions, respects exclusions, filters duplicates
  </done>
</task>
