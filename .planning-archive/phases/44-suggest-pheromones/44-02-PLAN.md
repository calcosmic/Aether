<task type="auto">
  <name>Task 1: Add suggest-approve command with tick-to-approve UI</name>
  <files>.aether/aether-utils.sh</files>
  <action>
    Add a `suggest-approve` command to aether-utils.sh that provides the tick-to-approve UI.

    The command should:
    1. Accept optional flags:
       - `--yes`: Auto-approve all suggestions without prompting
       - `--dry-run`: Show what would be approved without writing
       - `--no-suggest`: Skip suggestions entirely (for automated builds)
       - `--verbose`: Show detailed reasoning for each suggestion

    2. Call `suggest-analyze` to get the list of suggestions

    3. If no suggestions: Exit silently (no output)

    4. If suggestions exist, display header:
       ```
       ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
          S U G G E S T E D   P H E R O M O N E S
       ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

       Based on code analysis, the colony suggests these signals:
       ```

    5. For each suggestion, display one-at-a-time (adapted from learning-approve-proposals):
       ```
       ───────────────────────────────────────────────────
       Suggestion X of Y
       ───────────────────────────────────────────────────

       {emoji} {type} (priority: {priority}/10)

       {content}

       Detected in: {file}
       Reason: {reason}

       ───────────────────────────────────────────────────
       [A]pprove  [R]eject  [S]kip  [D]ismiss All  Your choice:
       ```

    6. Emoji per type:
       - FOCUS: 🎯
       - REDIRECT: 🚫
       - FEEDBACK: 💬

    7. Handle user input:
       - A/Approve: Write as FOCUS signal via `pheromone-write`
       - R/Reject: Dismiss, record hash to prevent re-suggestion
       - S/Skip: Dismiss without recording (may suggest again)
       - D/Dismiss All: Skip remaining suggestions
       - Invalid input: Default to Skip

    8. On Approve:
       - Call: `bash .aether/aether-utils.sh pheromone-write FOCUS "{content}" --source "system:suggestion" --reason "{reason}" --ttl "phase_end"`
       - Record hash via `suggest-record` to prevent duplicates
       - Display: "✓ Added FOCUS signal"

    9. On Reject/Skip:
       - Display: "→ Skipped" or "✗ Rejected"
       - Continue to next suggestion

    10. After all suggestions processed, display summary:
        ```
        ═══════════════════════════════════════════════════
        Summary: {approved} approved, {rejected} rejected, {skipped} skipped
        ═══════════════════════════════════════════════════
        ```

    11. Return JSON:
        ```json
        {
          "approved": N,
          "rejected": N,
          "skipped": N,
          "signals_created": ["sig_id_1", "sig_id_2"]
        }
        ```

    Handle non-interactive mode (no tty):
    - If stdin is not a tty and --yes not provided, exit silently (don't block)
    - Log: "Non-interactive mode: skipping suggestions (use --yes to auto-approve)"

    Reuse the UI patterns from learning-approve-proposals (lines 5057-5328).
  </action>
  <verify>
    bash .aether/aether-utils.sh suggest-approve --dry-run returns valid JSON with approved/rejected/skipped counts
  </verify>
  <done>
    suggest-approve command exists with tick-to-approve UI, handles all input options, returns JSON summary
  </done>
</task>
