  <action>
**The bug:** Inside markdown ```bash code blocks, commands end with `with description "some text"`. When Claude Code executes these blocks as literal bash, it fails with "with: command not found".

**The fix pattern:** Move the description OUT of the bash block into instruction prose above it. Transform:

```
\`\`\`bash
bash .aether/aether-utils.sh some-command "args" with description "Doing something..."
\`\`\`
```

Into:

```
Run using the Bash tool with description "Doing something...":
\`\`\`bash
bash .aether/aether-utils.sh some-command "args"
\`\`\`
```

**For the 757-char chained command in swarm.md (line ~147):** Break the 4 chained `swarm-display-update` calls into separate instructions, each with its own description line and bash block.

**Fix each Claude Code command file:**
- `swarm.md` — 13 instances (highest priority)
- `colonize.md` — 11 instances
- `entomb.md` — 11 instances
- `seal.md` — 8 instances
- `init.md` — 7 instances
- `plan.md` — 7 instances
- `pause-colony.md` — 1 instance

**Fix OpenCode counterparts:** Check each corresponding file at `.opencode/commands/ant/` for the same `with description` pattern inside bash code blocks. Fix any affected files using the identical pattern (description moved to prose above the bash block). If an OpenCode file does not contain the bug, skip it.

**Then add an AVA lint test** to `tests/unit/agent-quality.test.js` (or create a new `tests/unit/command-lint.test.js` if agent-quality.test.js is focused only on agents):

```javascript
test('CLEAN-03: no bash commands contain "with description" suffix', t => {
  // Scan .claude/commands/ant/*.md files
  // For each file, find lines inside ```bash blocks
  // Fail if any line matches: bash .* with description
  // This prevents the bug from reappearing
});
```

The test should scan both `.claude/commands/ant/` and `.opencode/commands/ant/` directories.

Important: Do NOT just add `\` line continuations. The fix is structural — description text belongs in instruction prose, not inside the bash command.
  </action>

3. `npm test` passes including the new lint test
