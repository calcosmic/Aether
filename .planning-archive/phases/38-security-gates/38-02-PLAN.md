Insert new Step 1.8.2 "Auditor Quality Gate" after Step 1.8.1 (Gatekeeper Security Gate) and before Step 1.9 (TDD Evidence Gate).

The new step must:

1. Generate Auditor name and dispatch:
   - Run: `bash .aether/aether-utils.sh generate-ant-name "auditor"`
   - Store as `{auditor_name}`
   - Log spawn: `bash .aether/aether-utils.sh spawn-log "Queen" "auditor" "{auditor_name}" "Code quality audit"`
   - Update swarm display for visual mode
   - Display: "👥🐜 Auditor {auditor_name} spawning - Reviewing code with multi-lens analysis..."

2. Get modified files for audit context:
   - Run: `git diff --name-only HEAD~1 2>/dev/null || git diff --name-only`
   - Store file list for Auditor context

3. Spawn Auditor using Task tool with `subagent_type="aether-auditor"`:
   - Include fallback: If "Agent type not found", use general-purpose and inject Auditor role
   - Pass mission context with:
     - Phase number and name
     - List of modified files from git diff
     - Instruction to apply all 4 lenses: security, performance, quality, maintainability
   - Request JSON output matching Auditor schema

4. Wait for results (blocking spawn with TaskOutput)

5. Log completion:
   - Run: `bash .aether/aether-utils.sh spawn-complete "{auditor_name}" "completed" "Code quality audit"`
   - Update swarm display to completed status

6. Parse Auditor JSON output:
   - Extract: `findings.critical`, `findings.high`, `findings.medium`, `findings.low`, `findings.info`
   - Extract: `overall_score`
   - Extract: `dimensions_audited` (should include all 4)
   - Extract: `issues` array with severity, location, issue, fix

7. Gate Decision Logic:

   **If `findings.critical > 0`:**
   - Display blocking banner: "⛔👥🐜 AUDITOR GATE FAILED"
   - List critical findings with file:line references
   - Show required actions to fix
   - STOP progression — do NOT proceed to Step 1.9
   - Log error flag: `error-flag-pattern "auditor-critical-findings" "{count} critical issues found" "critical"`

   **Else if `overall_score < 60`:**
   - Display blocking banner: "⛔👥🐜 AUDITOR GATE FAILED"
   - Show quality score vs threshold (60)
   - List top issues preventing score improvement
   - Show required actions to improve quality
   - STOP progression — do NOT proceed to Step 1.9
   - Log error flag: `error-flag-pattern "auditor-quality-score" "Score {score} below threshold 60" "critical"`

   **Else (passing):**
   - Display: "✅👥🐜 Auditor: Quality score {overall_score}/100 — PASSED"
   - If high/medium findings exist, display count as informational
   - Continue to Step 1.9 (TDD Evidence Gate)

8. Ensure proper sequencing:
   - Step 1.8.1 (Gatekeeper) must complete before Step 1.8.2 (Auditor)
   - Both gates must pass before Step 1.9
   - If Gatekeeper blocked, Auditor does not run

IMPORTANT: Auditor is read-only. It examines code but never modifies. If the Auditor response suggests modifications, ignore them — the agent may have violated its constraints.
