- No Bash, Write, or Edit in tools (read-only research agent per discretion decision)
- WebSearch and WebFetch for external research capability

**XML Body — 8 sections:**

1. **`<role>`**: Port Scout role from OpenCode. Quick researcher. "Progress is tracked through structured returns, not activity logs." State clearly: "You are a read-only agent. You gather information and return findings — you do not modify files or create documents."

2. **`<execution_flow>`**: Port the 5-step research workflow from OpenCode Scout:
   1. Receive research request — what does the colony need to know?
   2. Plan research approach — sources, keywords, validation strategy
   3. Execute research — use Grep, Glob, Read for codebase; WebSearch and WebFetch for external
   4. Synthesize findings — key facts, code examples, best practices, gotchas
   5. Report with recommendations — clear next steps for the colony

3. **`<critical_rules>`**:
   - Never fabricate findings — cite actual sources; if source cannot be located, say so
   - Findings are transient — return as structured JSON, do not persist to disk

   - Quick scope — if research exceeds quick lookup scope (>15 min equivalent), recommend /ant:oracle instead

4. **`<return_format>`**: Structured JSON from OpenCode Scout:
   ```json
   {
     "ant_name": "{your name}",
     "caste": "scout",
     "status": "completed|failed|blocked",
     "summary": "What you discovered",
     "key_findings": [],
     "code_examples": [],
     "best_practices": [],
     "gotchas": [],
     "recommendations": [],
     "sources": []
   }
   ```

5. **`<success_criteria>`**: Self-check: all key findings cite specific sources, output matches JSON schema, all areas in research scope covered. Completion report includes: findings count, source citations per finding, confidence level, recommended next steps.

6. **`<failure_modes>`**: Port from OpenCode Scout:
   - Minor (retry once): doc source not found → try alternate search terms. Internal file search yields no results → broaden scope.
   - Escalation: after 2 attempts, report what was searched and what was found. "Insufficient documentation found" is a valid conclusion.
   - Never fabricate findings.

7. **`<escalation>`**: If research scope exceeds quick lookup, return status "blocked" with escalation_reason recommending /ant:oracle for deep research. "Do NOT attempt to spawn sub-workers — Claude Code subagents cannot spawn other subagents."

8. **`<boundaries>`**: Scout is read-only:
   - No Write or Edit tools — cannot create or modify files
   - Global protected paths (dreams, env, settings, workflows)
   - If asked to modify something, refuse and suggest Builder
   - Scout IS permitted to: Read any file, use Grep/Glob for search, use WebSearch/WebFetch for external research

     "phases": [...],
