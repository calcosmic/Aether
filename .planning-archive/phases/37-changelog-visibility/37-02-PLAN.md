## User Decisions (LOCKED)

### CHANGELOG.md Format
- Organized by date, then phase: `## 2026-02-21` with `### Phase 36` subsections under each date
- Not flat log, not purely hierarchical by phase

### Changelog Update Timing
- After each plan completes (not after each task, not after each phase)
- One entry per plan, aggregating all decisions/files from that plan

### Changelog Content (each entry includes)
- Files changed (modified, added, deleted)
- Decisions made during the plan (why this approach)
- What worked/didn't (from Phase 36 midden/failures system)
- Requirements addressed by this plan

### Claude's Discretion
- Changelog file location: Using root CHANGELOG.md (standard convention)
- Handling existing Keep a Changelog format compatibility

## Existing CHANGELOG.md Format

The existing CHANGELOG.md uses Keep a Changelog format with version headers:
```markdown
## [Unreleased]
### Changed
- README updated

## v5.0.0 — Worker Emergence (2026-02-20)
### Added
- 22 Claude Code subagents
```

New entries will be appended after existing content with a clear separator.

## Data Sources for Changelog Entries

1. **Files changed**: From plan frontmatter files_modified
2. **Decisions**: From COLONY_STATE.json memory.decisions (last 5)
3. **What worked/didn't**: From midden.json failure entries
4. **Requirements**: From plan frontmatter requirements field
</context>

  <name>Task 1: Create changelog-append function</name>

  <action>
    Add a new function `changelog-append` to aether-utils.sh that appends entries to CHANGELOG.md with the date-phase hierarchy format.

    Function signature:
    ```bash
    changelog-append <date> <phase> <plan> <files> <decisions> <worked> <requirements>
    ```

    Parameters:
    - date: ISO date string (e.g., "2026-02-21")
    - phase: Phase identifier (e.g., "36-memory-capture")
    - plan: Plan number (e.g., "01")
    - files: Comma-separated list of files changed
    - decisions: Semicolon-separated list of decisions
    - worked: What worked/didn't (semicolon-separated)
    - requirements: Comma-separated list of requirement IDs

    The function should:
    1. Check if CHANGELOG.md exists, create if not
    2. Check if date section (## YYYY-MM-DD) exists
       - If not, append new date section
    3. Append phase subsection (### Phase X) under the date
    4. Add entry with all fields in consistent format

    Entry format:
    ```markdown
    ### Phase 36 — Plan 01

    - **Files:** `file1.md`, `file2.sh` (modified); `new.md` (added)
    - **Decisions:** Used approach X because Y; Deferred Z to future phase
    - **What Worked:** Pattern A succeeded; Approach B was faster than expected
    - **What Didn't:** Approach C failed due to D (logged to midden)
    - **Requirements:** MEM-01, MEM-02 addressed
    ```

    Handle special characters in inputs (escape markdown, handle empty fields).

    Insert in the command dispatch section near other utility functions.
  </action>
  <verify>
    bash .aether/aether-utils.sh changelog-append "2026-02-21" "36-memory-capture" "01" "file1.md,file2.sh" "Decision A;Decision B" "Pattern X worked" "MEM-01,MEM-02"
    grep -A 10 "## 2026-02-21" CHANGELOG.md | grep -q "Phase 36"
    # Should find the new entry
  </verify>
  <done>changelog-append function creates properly formatted entries in CHANGELOG.md</done>
