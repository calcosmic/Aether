<task type="auto">
  <name>Create parse_selection helper function</name>
  <files>.aether/aether-utils.sh</files>
  <action>
    Add a new helper function `parse_selection()` to aether-utils.sh that:
    1. Takes arguments: input_string max_index
    2. Outputs space-separated 0-indexed indices (e.g., "0 2 4")
    3. Validates each number is within range 1-max_index
    4. Deduplicates selections (if user enters "1 1 2", output "0 1")
    5. Returns empty string if input is empty (defer all)
    6. Skips invalid numbers with warning to stderr (don't fail)
    7. Normalizes input: remove extra spaces, keep only digits and spaces
    8. Uses 1-indexed input (user types 1, 2, 3) converted to 0-indexed output

    Handle edge cases:
    - Empty input -> return 1 (defer all signal)
    - All invalid numbers -> return error
    - Mixed valid/invalid -> return valid, warn about invalid

    Place this function near other helpers (after generate_threshold_bar from 34-01).
    Use the pattern from 34-RESEARCH.md Pattern 1 and Example 2.
  </action>
  <verify>
    source .aether/aether-utils.sh
    parse_selection "1 3 5" 10
    # Should output: "0 2 4"
    parse_selection "" 5
    # Should return 1 (empty/defer signal)
    parse_selection "1 2 99" 5
    # Should output "0 1" and warn about 99 being invalid
  </verify>
  <done>
    Function correctly parses space-separated numbers
    Converts 1-indexed to 0-indexed
    Validates range and warns on invalid
    Deduplicates selections
    Returns empty/defer signal for empty input
  </done>
</task>

    9. Shows summary: "3 proposals selected, 2 deferred"

    The function should:
    - Handle the display -> input -> parse flow
    - Return structured JSON for downstream processing
    - Be usable both interactively and programmatically
    - Support --dry-run to show what would be selected without capturing input

    Add this case after learning-display-proposals.
  </action>

    # Should output defer_all action with all indices in deferred
  </verify>
  <done>
