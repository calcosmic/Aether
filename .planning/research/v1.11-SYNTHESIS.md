## Executive Summary

v1.11 has four workstreams: (1) remove self-hosting artifacts from Aether-developing-Aether, (2) restore lost intelligence features from the April 2026 shell-to-Go migration, (3) harden the 3-platform experience, and (4) improve user-facing flows. All work uses existing Go runtime patterns — zero new dependencies.

## Key Findings

### Self-Hosting Artifacts (Agent Exploration)

The research agents found concrete self-hosting artifacts:

1. **`.aether/agents/` vs `.aether/agents-claude/`**: All 26 agent files differ between the two directories. Per CLAUDE.md, `agents-claude/` is the packaging mirror that should be byte-identical to `.claude/agents/ant/`. The `agents/` directory is a stale third copy with outdated content.

2. **241 tracked chamber files in `.aether/chambers/`**: Old colony entomb artifacts from self-hosting sessions. These bloat the repo and should be gitignored or removed.

3. **Self-hosting state files tracked in git**: `.aether/CONTEXT.md`, `.aether/CROWNED-ANTHILL.md`, `.aether/QUEEN.md` are tracked but are runtime-generated state from colony sessions.

4. **Orphaned chambers**: 16 chamber directories from past self-hosting sessions, containing stale colony state, dreams, and handoff files.

### Platform Gaps

- **OpenCode parity**: Missing shelf sections in init.md and entomb.md (from v1.10 audit)
- **Codex subagent dispatch**: The current branch (`codex/fix-opencode-subagent-dispatch`) suggests active work on Codex platform issues
- **120+ CLI flag mismatches**: Previous audit found markdown wrappers using wrong flags against Go CLI

### Recommended Approach

**Phase 1: Self-hosting cleanup** (lowest risk, immediate value)
- Remove `.aether/agents/` stale directory
- Add chambers/ to .gitignore, remove tracked chamber files
- Remove or gitignore runtime state files (CONTEXT.md, CROWNED-ANTHILL.md, QUEEN.md)
- Audit for other self-hosting artifacts

**Phase 2: Platform hardening**
- Fix OpenCode shelf parity gaps
- Fix CLI flag mismatches in wrapper commands
- Verify Codex subagent dispatch

**Phase 3: Smart Init restoration**
- Port charter ceremony to Go
- Port rich init-research to Go

**Phase 4: Intelligence features**
- Port suggest-analyze to Go
- Restore Bayesian confidence scoring
- Restore circuit breaker

**Phase 5: UX improvements**
- Improve onboarding flows
- Clearer feedback and error messages
- Smoother ceremony flows

### Risks

1. **Restoring old shell code**: The deleted shell scripts may reference patterns that don't fit the Go architecture. Port, don't copy.
2. **Chamber deletion**: Removing 241 tracked files is a large git operation. Use `git rm` carefully.
3. **Init ceremony complexity**: The charter approval flow requires interactive prompts. Go cobra supports this but needs careful UX design.
4. **Feature prioritization**: Not all 9 lost features are equally valuable. Suggest-analyze and Smart Init are highest user impact.
