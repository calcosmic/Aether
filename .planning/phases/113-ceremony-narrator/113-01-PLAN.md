    Create `.aether/ts-host/src/template-loader.ts` with the following exported API:

    1. `parseTemplate(raw: string): { frontmatter: Record<string, unknown>; body: string }`
       - Parse YAML frontmatter delimited by `---\n` ... `\n---\n`.
       - Throw `Error("Invalid template: missing YAML frontmatter")` if delimiter missing.
       - Use `js-yaml` (already in package.json) to parse frontmatter.
       - Return body as the remainder after the second `---`.

    2. `substituteTemplate(body: string, vars: Record<string, string>): string`
       - Replace `/{(\w+)(?::([^}]*))?}/g` with `vars[key] ?? fallback ?? ""`.

    3. `loadTemplate(cwd: string, name: string): { frontmatter: Record<string, unknown>; body: string }`
       - Resolve path: `join(cwd, ".aether", "templates", "ceremony", "${name}.md")`.
       - Read file with `readFileSync` in utf-8.
       - If file missing, look up `name` in `DEFAULT_TEMPLATES` (inline defaults map) and return that object.
       - If neither file nor default exists, throw `Error("Template not found: ${name}")`.
       - Call `parseTemplate` on the raw string and return result.

    4. `DEFAULT_TEMPLATES: Record<string, { frontmatter: Record<string, unknown>; body: string }>`
       - Inline fallback defaults for at least these names:
         - `banner-build-start` — frontmatter: `{ figlet_font: "Standard", emoji: "🔨", title: "BUILD" }`, body: `\n{figlet_banner}\n\n── {stage} ──\n\n{content}\n`

         - `spawn-frame` — frontmatter: `{}`, body: `{emoji} {label} {name}  {task}\n`
         - `stage-separator` — frontmatter: `{}`, body: `{prefix}{stage}{suffix}\n`
         - `build-summary` — frontmatter: `{ borderStyle: "round", borderColor: "green" }`, body: `\n{content}\n`
         - `closeout-ritual` — frontmatter: `{ borderStyle: "double", borderColor: "cyan" }`, body: `\n{content}\n`
       - These defaults must be literal strings in the source file (not loaded from disk).

    Imports: `readFileSync` from `node:fs`, `join` from `node:path`, `yaml` from `js-yaml`.
    Use `.js` extension for local imports (project convention: ESM + NodeNext).
