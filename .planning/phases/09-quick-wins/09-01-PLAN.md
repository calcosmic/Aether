**REL-03: Learning-observations circuit breaker (aether-utils.sh)**

In `.aether/aether-utils.sh`, the `learning-observe` case starting at line 5318, replace the hard error on corrupt JSON with a recovery path:

1. Before the JSON validation block (line 5318-5321), add a rolling backup rotation. Before each write to observations_file (there are two write paths — existing observation update at ~line 5351 and new observation at ~line 5376), add:
   ```
   # Rotate backups before write
   if [[ -f "$observations_file" ]]; then
     cp -f "${observations_file}.bak.2" "${observations_file}.bak.3" 2>/dev/null || true
     cp -f "${observations_file}.bak.1" "${observations_file}.bak.2" 2>/dev/null || true
     cp -f "$observations_file" "${observations_file}.bak.1" 2>/dev/null || true
   fi
   ```

2. Replace the `json_err` call at line 5320 with recovery logic (see RESEARCH.md Pattern 3 for exact code):
   - Try to recover from `.bak.1`, `.bak.2`, `.bak.3` in order
   - If a valid backup is found: restore it and emit `echo "⚠ Learning observations file was corrupted — restored from backup. Some recent entries may be missing." >&2`
   - If backups exist but ALL are corrupted: call `json_err` with message "Learning observations and all 3 backups are corrupted. Manual recovery needed."
   - If NO backups exist (first-time corruption): emit `echo "⚠ Learning observations file was corrupted. Starting fresh — this is a first-time recovery." >&2` and reset to `{"observations":[]}`

Per user decision: warn before recovering if data loss is involved. If main + all backups corrupted, stop (don't auto-reset).
