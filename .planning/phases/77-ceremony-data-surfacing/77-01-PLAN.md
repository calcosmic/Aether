# Key interfaces from existing codebase

From cmd/init_research.go -- the JSON envelope output by init-research contains:
```go
outputOK(map[string]interface{}{
    "tech_stack_detail":      techStackDetail,      // []techStackDetail
    "dir_classification":     dirClass,             // dirClassification
    "governance_details":     governanceDetails,    // []governanceDetail
    "colony_context_summary": contextSummary,       // colonyContextSummary
})
```

From cmd/init_ceremony.go -- runCeremonyResearch already extracts charter and pheromone_suggestions from the envelope but ignores the 4 research fields above. The function returns (*colony.Charter, []pheromoneSuggestion, error).

From cmd/codex_visuals.go -- renderCharterDisplay renders 7 charter sections. A new renderResearchDisplay function will follow the same pattern.

From cmd/circuit_breaker.go -- three emit functions use fmt.Printf:
- emitCircuitBreakerTripped: printf on trip
- emitCircuitBreakerRedistributed: printf on redistribution
- emitCircuitBreakerNoPeer: printf when no peer available

From cmd/ceremony_emitter.go -- emitBuildCeremonyCircuitBreak exists and is properly wired to the event bus but is NEVER CALLED. It accepts (phase, wave, CircuitBreakerEvent).

From cmd/codex_build_worktree.go -- the dispatch loops call cb.RecordFailure but only call emitCircuitBreakerRedistributed/emitCircuitBreakerNoPeer (printf). They do NOT call emitCircuitBreakerTripped or emitBuildCeremonyCircuitBreak.

From cmd/codex_workflow_cmds.go -- buildCmd flags do NOT include --no-suggest. The build-context.md playbook references it.

From .aether/docs/command-playbooks/build-context.md Step 4.2:
"Conditional step -- skipped if --no-suggest flag is passed."
"If suggest_enabled is false (from --no-suggest flag): Skip to Step 4.3."
