Create `cmd/parity_test.go` in package `cmd` with the following structure:

1. **ParitySnapshot struct** for the golden file:
```go
type ParitySnapshot struct {
    YAMLCatalog           []string            `json:"yaml_catalog"`
    ClaudeWrapperCatalog  []string            `json:"claude_wrapper_catalog"`
    OpenCodeWrapperCatalog []string           `json:"opencode_wrapper_catalog"`
    CommandGuideCatalog   []string            `json:"command_guide_catalog"`
    RuntimeCatalogNames   []string            `json:"runtime_catalog_names"`
    RuntimeResolvedNames  map[string]string   `json:"runtime_resolved_names"`
}
```
Each `*_Catalog` field is a sorted []string of command names from that surface. `RuntimeResolvedNames` maps YAML command names to their Cobra primary names (only for entries where they differ).

2. **Alias resolution map** -- copy this verified map directly into the test file:
```go
var yamlToRuntimeName = map[string]string{
    "export-signals":  "pheromone-export-xml",
    "import-signals":  "pheromone-import-xml",
    "flag":            "flag-add",
    "flags":           "flag-list",
    "insert-phase":    "phase-insert",
    "memory-details":  "memory-metrics",
    "patrol":          "patrol-check",
    "pheromones":      "pheromone-display",
    "profile":         "profile-read",
    "resume":          "resume-colony",
    "shelf":           "shelf-list",
}
```

3. **Exclusion sets:**
```go
var promptOnlyCommands = map[string]bool{
    "archaeology": true, "chaos": true, "dream": true,
    "interpret": true, "organize": true,
}
var cobraBuiltinCommands = map[string]bool{
    "help": true,
}
```

4. **Surface extraction functions:**
- `extractYAMLNames(t) []string` -- reuse `yamlCommandNamesForGuideTest(t)` pattern: read `.aether/commands/` directory, filter `.yaml` files, trim suffix, sort
- `extractWrapperNames(t, repoRoot, subdir) []string` -- read directory entries from `subdir` (e.g., `.claude/commands/ant`), filter `.md` files, trim suffix, sort
- `extractCommandGuideNames() []string` -- call `commandGuideCatalog()`, collect keys into sorted slice
- `extractRuntimeNames() []string` -- call `buildAuditCatalog(rootCmd)`, collect `entry.Name` into sorted slice

5. **TestPlatformParityGolden** function:
- Build a `ParitySnapshot` by calling all 4 extraction functions
- Build `RuntimeResolvedNames` by iterating YAML names and looking up in `yamlToRuntimeName` (only include entries that map)
- Marshal to JSON with indent
- If `*updateGolden` flag is set: write to `testdata/parity_snapshot.json` and return
- Otherwise: read golden file and compare. On mismatch, error with "run with -update-golden to refresh"

6. **TestNoPhantomCommands** function (per D-03 and PLAT-03):
- Build runtime name set from `buildAuditCatalog(rootCmd)`
- Build YAML name set
- For each YAML name NOT in `promptOnlyCommands` and NOT in `cobraBuiltinCommands`:
  - Resolve to runtime name using `yamlToRuntimeName` (or use the YAML name directly if not in map)
  - Assert the resolved name exists in the runtime name set
  - If missing, report as phantom command
- For each Claude wrapper name: same check (after alias resolution)
- For each OpenCode wrapper name: same check (after alias resolution)
- For each command-guide name: if not in `promptOnlyCommands`/`cobraBuiltinCommands`, resolve and assert present in runtime
- This test MUST pass with the current codebase -- any known phantoms should be in the exclusion sets

7. **TestAllYamlHaveWrappersAndGuide** function (extends existing parity checks per PLAT-02):
- For each YAML name: assert a matching Claude wrapper `.md` file exists
- For each YAML name: assert a matching OpenCode wrapper `.md` file exists
- For each YAML name: assert the name exists in `commandGuideCatalog()` keys
- For each Claude/OpenCode wrapper name not in YAML set: report as orphan wrapper
- This test MUST pass with current codebase

8. **TestAliasResolutionCompleteness** function:
- For each entry in `yamlToRuntimeName`: assert the resolved runtime name actually exists in `buildAuditCatalog(rootCmd)` output
- This catches stale alias entries if runtime commands are renamed

9. After writing the test file, run with `-update-golden` to create the initial golden file:
```bash
go test ./cmd/ -run TestPlatformParityGolden -update-golden -count=1
```

10. Then verify all new tests pass:


11. Then verify the full test suite still passes:
