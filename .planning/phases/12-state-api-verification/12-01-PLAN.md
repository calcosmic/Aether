Create `.aether/utils/state-api.sh` following the proven hive.sh/midden.sh extraction pattern. The module header should match the hive.sh convention:

```bash
#!/bin/bash
# State API facade -- centralized COLONY_STATE.json access
# Provides: _state_read, _state_write, _state_read_field, _state_mutate
#
# These functions are sourced by aether-utils.sh at startup.
# All shared infrastructure (json_ok, json_err, atomic_write, acquire_lock,
# release_lock, LOCK_DIR, DATA_DIR, SCRIPT_DIR, error constants) is available.
```

Implement four core functions based on the patterns in the research doc (Pattern 1):

1. **_state_read** -- Read full COLONY_STATE.json. Validate JSON before returning. No lock needed for reads (jq is atomic on single files). Return via `json_ok`. Error via `json_err` with `$E_FILE_NOT_FOUND` or `$E_JSON_INVALID`.

2. **_state_read_field** -- Read a specific jq field path (e.g., `.current_phase`). For internal callers, output the raw value to stdout (no json_ok wrapper). For the subcommand case entry (`state-read-field`), wrap in json_ok. Follow the hive.sh pattern where internal functions return raw and case entries wrap. Handle missing field by returning empty string and exit 0 for internal use (callers check emptiness).

3. **_state_write** -- Refactor the existing state-write case block (aether-utils.sh lines 1297-1332) into this function. The logic is: validate JSON input, acquire lock, create_backup, atomic_write, release lock, json_ok. The existing state-write case entry becomes a thin delegation: `state-write) _state_write "$@" ;;`. This preserves backward compatibility.

4. **_state_mutate** -- New function for read-modify-write operations. Takes a jq expression. Acquires lock, creates backup, applies jq expression to current state, validates result, writes via atomic_write, releases lock. Uses `json_err` with `$E_LOCK_FAILED`, `$E_JSON_INVALID`, `$E_UNKNOWN` for error paths. Follow the research code example closely.

Also add **_state_migrate** -- move the migration logic from the validate-state case block (around line 1180 in aether-utils.sh) into state-api.sh. The validate-state case should call `_state_migrate` instead of having inline migration logic.

Wire into aether-utils.sh:
1. Add sourcing line after the skills.sh source (line 34 area): `[[ -f "$SCRIPT_DIR/utils/state-api.sh" ]] && source "$SCRIPT_DIR/utils/state-api.sh"`
2. Add case entries for the new subcommands:
   - `state-read) _state_read "$@" ;;`
   - `state-read-field) _state_read_field "$@" ;;`
   - `state-mutate) _state_mutate "$@" ;;`
3. Refactor the existing `state-write)` case to delegate: `state-write) _state_write "$@" ;;`
4. Update the existing `validate-state)` case to call `_state_migrate` if migration logic was moved

Add `# MIGRATE: direct COLONY_STATE.json access -- use _state_read_field instead` comments on the existing state-write case block location (now just a delegation) as a breadcrumb.
