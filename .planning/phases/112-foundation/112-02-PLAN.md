    Create `.aether/ts-host/src/event-bridge.ts` with the following exports:
    1. `EventBridgeOptions` interface extending `GoBridgeOptions` with optional `filter` (default "ceremony.*"), `pollIntervalMs` (default 250), `replaySince` (ISO string, default empty), and `onEvent` callback.
    2. `startEventBridge(opts)` async function that:
       a. Calls `aether event-bus-replay --topic <filter> --since <replaySince>` via `callGoJSON` to get historical events.
       b. Spawns `aether event-bus-subscribe --stream --filter <filter> --poll-interval <pollIntervalMs>ms` as a long-running child process using `spawn` from `node:child_process`.
       c. Parses NDJSON lines from stdout, validates each line is a valid `CeremonyEvent`, and emits via `onEvent` callback.
       d. Tracks seen event IDs in a Set to deduplicate replay->stream handoff.
       e. Returns a controller object with `stop()` method that kills the subprocess and clears the seen set.
    3. `stopEventBridge(controller)` convenience function that calls `controller.stop()`.
    4. A runtime boundary guard: if any code path attempts to open a file under `.aether/data/` in write mode (including `event-bus.jsonl`), throw `BoundaryViolationError` with message "TS host attempted to write to Go-owned path: {path}".
    5. The event bridge MUST only read from `.aether/data/event-bus.jsonl` (never write). It must NOT use `fs.createWriteStream`, `writeFileSync`, `appendFileSync`, or any write API on paths under `.aether/data/`.
    Use `readline` from `node:readline` to process NDJSON lines from the subprocess stdout. Handle subprocess stderr by logging to `process.stderr` with prefix `[event-bridge]`.
  </action>
  <verify>
    <automated>cd .aether/ts-host && npm run typecheck</automated>
  </verify>

    - Uses `spawn` for `event-bus-subscribe --stream` and `callGoJSON` for replay
    - Deduplicates events via ID Set
    - Contains runtime boundary guard rejecting write mode on `.aether/data/` paths
  </done>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TS host -> Go subprocess | NDJSON from Go CLI is parsed as JSON; malformed input could crash parser |
| TS host -> .aether/data/ | Read-only boundary; write attempts must be rejected |
| YAML config -> TS host | Untrusted file content parsed at runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-112-04 | Spoofing | event-bridge.ts NDJSON parser | mitigate | Validate parsed object has required Event fields (id, topic, payload) before casting to CeremonyEvent |
| T-112-05 | Tampering | event-bridge.ts subprocess | accept | Go CLI is trusted; subprocess stdout is not attacker-controlled in local dev context |
| T-112-06 | Denial of Service | event-bridge.ts memory growth | mitigate | Limit seen-event Set size to 10000 entries with LRU eviction; document in code |
| T-112-07 | Elevation of Privilege | boundary-reference.ts write guard | mitigate | Runtime check throws on any write mode to .aether/data/; test coverage enforces this |
| T-112-08 | Denial of Service | caste-config.ts YAML parser | mitigate | Use js-yaml safeLoad (not load); validate required keys after parse |
</threat_model>
