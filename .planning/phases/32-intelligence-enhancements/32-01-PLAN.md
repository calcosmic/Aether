  <files>.aether/utils/scan.sh</files>
  <action>
Add two new sub-scan functions to scan.sh, following the exact same pattern as existing functions (_scan_tech_stack, _scan_prior_colonies, etc.): each takes a repo root argument, returns raw JSON via stdout, and uses jq -n for assembly.

**_scan_colony_context(repo_root):**
- Read chambers from `$root/.aether/chambers/`
- For each chamber (most recent first, max 3), extract:
  - From `manifest.json`: goal, phases_completed, total_phases, milestone
  - From `CROWNED-ANTHILL.md` (if exists): extract "The Work" section (use `sed -n '/^## The Work$/,/^## /p'` then strip headers, take first 2 lines, join). This is the summary.
  - IMPORTANT: Do NOT read COLONY_STATE.json from chambers (15-20KB). Only use manifest.json and CROWNED-ANTHILL.md.
  - If neither manifest.json nor CROWNED-ANTHILL.md exist, skip that chamber.
- Also extract existing charter content from `$root/.aether/QUEEN.md` (if it exists):
  - Use grep for `[charter] **Intent**:`, `[charter] **Vision**:`, `[charter] **Governance**:` patterns
  - Strip `(Colony: ...)` suffixes with sed
  - If no charter entries found, return empty strings
- Sort chambers by directory name descending (names start with dates, so alphabetical reverse = most recent first)
- Return JSON: `{prior_colonies: [{goal, phases, outcome, summary}...], existing_charter: {intent, vision, governance}}`
- When no chambers directory exists or no valid chambers found, return `{prior_colonies: [], existing_charter: {intent: "", vision: "", governance: ""}}`

**_scan_governance(repo_root):**
- Detect governance-related config files and produce prescriptive rules
- Categories to check (with cross-reference validation per research pitfall 2):
  1. **CONTRIBUTING.md**: If exists, extract first 20 lines for summary (head -20, join, truncate to 200 chars). Rule: "Follow CONTRIBUTING.md guidelines". Source: "CONTRIBUTING.md".
  2. **Test configs**: Check for jest.config.*, vitest.config.*, pytest.ini, pyproject.toml, Cargo.toml with [test], go test files. ONLY emit "TDD required" if test FILES also exist (check for *.test.*, *.spec.*, test_* files in tests/ or __tests__/ dirs). Source: "test configuration".
  3. **Linter/formatter configs**: Check for .eslintrc.*, eslint.config.*, .prettierrc*, rustfmt.toml, .flake8, pyproject.toml [tool.black]. Emit rule per detected tool. Source: tool name.
  4. **CI/CD**: Check for .github/workflows/, Jenkinsfile, .gitlab-ci.yml, .circleci/. Emit "CI/CD pipeline active -- ensure all checks pass before merging". Source: "CI configuration".
- Each rule is JSON: `{rule: "...", source: "...", strength: "required"}`. Detail field only for CONTRIBUTING.md.
- Governance should focus on PROCESS and STANDARDS, not technology choices (per pitfall 6).
- Skip silently if config file exists but has no meaningful content (per user decision on weak/boilerplate files).
- Return JSON: `{rules: [...], sources_checked: N}`
