  <action>
    Create scripts/smoke-test-classic.sh as an executable Bash script. Per D-05, this is a standalone Bash script (not a Go test).

    The script must follow this structure:

    **Preamble:** shebang `#!/usr/bin/env bash`, `set -euo pipefail`. Define CLASSIC_TAG="v5.4.0" and WORKDIR="". Define a cleanup function that removes the git worktree if it exists. Trap cleanup on EXIT.

    **Setup:** Create a temp directory with mktemp -d. Run `git worktree add "$WORKDIR" "$CLASSIC_TAG" --detach` to check out v5.4.0 in isolation (per Pitfall 4 -- never modify working tree). cd into WORKDIR.

    **Isolation for delegation shim (per Pitfall 2):** Set HOME to a temp directory so v5.4.0's version-gate.js cannot find ~/.aether/bin/aether. This prevents the delegation shim from delegating to the Go binary. Create a temp dir and export HOME to it.

    **npm install:** Run `npm install --production --silent`. Per Pitfall 5, this is required because node_modules is not committed in the v5.4.0 tag. Dependencies are commander, js-yaml, picocolors.

    **Test 1 - CLI help:** Run `node bin/cli.js --help` and verify exit code 0.

    **Test 2 - Colony init:** Run `node bin/cli.js init --goal "smoke-test"`. Verify exit code 0. Verify .aether/data/COLONY_STATE.json was created. Per D-04, this verifies state creation works.

    **Test 3 - State structure:** Read COLONY_STATE.json and verify it contains the fields: "version", "current_phase", "events". Use grep for field presence (no need for jq).

    **Test 4 - State mutation across commands (per D-04):** Capture the SHA-256 hash of COLONY_STATE.json before sync-state. Run `node bin/cli.js sync-state`. Capture the hash after sync-state. Verify the hashes differ -- this proves COLONY_STATE.json changes between CLI invocations, satisfying D-04's requirement for observing state mutation across commands. Print the before/after hashes on PASS.

    **Test 5 - Wrapper commands with ceremony content (per D-04):** Verify the wrapper Markdown files exist at .claude/commands/ant/ for: plan.md, build.md, continue.md, init.md. Per D-04, since plan/build/continue are slash commands (not CLI subcommands), the test verifies wrapper file presence AND checks that they contain ceremony-related content. For each wrapper file, grep for at least one ceremony marker: "Stage", "worker", "caste", or "Builder" (case-insensitive). Report which markers were found.

    **D-04 Deviation Acknowledgment:** The smoke test cannot execute plan/build/continue as CLI lifecycle commands because Classic never implemented them as CLI subcommands -- they were slash commands only (Markdown wrappers executed by the AI platform). This is an architectural limitation of Classic, not a test gap. The test verifies the maximum that Classic can provide: (1) CLI init creates and mutates state, (2) sync-state causes observable state mutation between CLI invocations, (3) wrapper files exist with ceremony/caste content markers. The full plan/build/continue lifecycle test with ceremony stage markers in actual output belongs in Phase 108 (Golden Workflow Tests) against the Go runtime, where these are real subcommands. Add a comment in the script documenting this deviation.

    **Test 6 - Module inventory:** Verify all 16 lib modules exist at bin/lib/: banner.js, binary-downloader.js, caste-colors.js, colors.js, errors.js, event-types.js, file-lock.js, init.js, interactive-setup.js, logger.js, nestmate-loader.js, spawn-logger.js, state-guard.js, state-sync.js, update-transaction.js, version-gate.js.

    **Test 7 - Version output:** Run `node bin/cli.js version` and verify it outputs something (exit code 0). With HOME isolated, this should show the Node package version.

    **Test 8 - Status command:** Run `node bin/cli.js status` and verify exit code 0 (colony was initialized in Test 2).

    Each test must print "PASS: {test name}" on success and "FAIL: {test name}" with details on failure. Script exits with code 1 on first failure.

    At the end, print "=== All smoke tests passed ===" with a count.

    Make the script executable (chmod +x).

    CRITICAL: Do NOT use `git stash` or `git checkout v5.4.0` -- only use `git worktree add`. The script must never modify the working tree.
