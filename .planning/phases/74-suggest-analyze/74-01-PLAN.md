The command structure:
- Use: "suggest-analyze"
- Args: cobra.NoArgs
- Flags: --dry-run (bool), --target (string, default ".")

Command logic (in RunE):
1. Store nil guard (standard pattern)
2. Load active colony state via loadActiveColonyState(). If error, return ok:true with empty suggestions (non-blocking per RESEARCH Pitfall 3)
3. Check --no-suggest is handled at the playbook level, not here -- the command itself always runs when called
4. Resolve target directory from flag (default ".")
5. CHANGE DETECTION (per D-01 -- threshold-based, NOT exact HEAD match):
   - Get current git HEAD commit hash via `exec.Command("git", "-C", target, "rev-parse", "HEAD").Output()`
   - If cs.LastAnalyzeCommit is nil (first build): proceed with full analysis
   - If cs.LastAnalyzeCommit is not nil: run git diff to count changed files:
     ```go
     out, err := exec.Command("git", "-C", target, "diff", "--stat", *cs.LastAnalyzeCommit, "HEAD").Output()
     ```
   - Count the number of lines in the diff --stat output (each line represents one changed file). Use strings.Count(out, "\n") or split and count non-empty lines.
   - Define a constant: `const changeThreshold = 5` (Claude's discretion per CONTEXT.md)
   - If changed file count is BELOW the threshold (including 0 changes): skip analysis, return ok:true with existing pending suggestions from colony state
   - If changed file count meets or exceeds the threshold: proceed with analysis
   - This implements D-01's "changed significantly" semantics: small changes (under 5 files) do not trigger re-analysis, but significant changes do. If the user makes changes and commits them, only re-analysis triggers if enough files changed -- matching D-01's intent of threshold-based filtering.
6. Run detectGovernance(target) to get governance info
7. Run classifyDirectory(target) to get directory classification
8. Run parseDependencyFiles(target) to get tech stack details
9. Call generatePheromoneSuggestions(target, governance, dirClass, techStack) to get the 25 base suggestions
10. Run BUILD-SPECIFIC EXTRA PATTERNS (per D-02):
   a. TODO/FIXME density: count occurrences of "TODO" and "FIXME" across codebase files using `fileContains()` on known file types. If > 10 occurrences, add FEEDBACK suggestion: "high TODO/FIXME density (N found) -- consider addressing tech debt"
   b. Large files: iterate files in target, check if any Go files exceed 500 lines using `readFileContent()` with line counting. If found, add FEEDBACK: "large file detected (filename) -- consider splitting"
   c. Test gaps: check if directories containing Go source files have corresponding `_test.go` files. For any directory with > 3 .go files but no _test.go files, add FEEDBACK: "no tests found in (directory)"
   d. New dependencies: if tech stack has > 20 direct dependencies, add FEEDBACK: "high dependency count (N) -- review for necessity"
11. DEDUPLICATION (per D-07, D-08):
   - Load pheromones.json via store.LoadJSON("pheromones.json", &pf)
   - Build a set of (type, contentHash) from ACTIVE signals only (skip inactive/expired per D-08)
   - Filter out any suggestion whose type matches and whose content SHA-256 matches an active signal's content hash
   - Content hash for comparison: "sha256:" + sha256Sum(suggestion.Content)
12. SANITIZE each remaining suggestion's content via colony.SanitizeSignalContent(). Skip any that fail sanitization.
13. PERSIST to colony state:
   - Create PendingSuggestion structs with ID (use generateSignalID()), Type, Content, Reason, ContentHash, CreatedAt, Dismissed=false
   - Merge with existing pending suggestions: keep any that aren't in the new set (by content hash comparison). Replace the full list.
   - If --dry-run, skip this step -- return suggestions without persisting
   - Save updated colony state via store.SaveJSON("COLONY_STATE.json", cs)
   - Update LastAnalyzeCommit to current HEAD commit hash
14. Return outputOK with:
   ```go
   map[string]interface{}{
       "suggestions": filteredSuggestions,
       "total":       len(filteredSuggestions),
       "new_count":   newCount,
       "skipped_dedup": skippedCount,
       "dry_run":     dryRun,
   }
   ```
