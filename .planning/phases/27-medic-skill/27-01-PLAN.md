    3. Document the following sections:

       **## Colony Data Health Reference**
       - Purpose statement: "This skill defines what healthy colony data looks like. Workers should reference this when modifying colony state files."

       **## COLONY_STATE.json Healthy State**
       - Schema: version "3.0", goal (string, required), state (IDLE|READY|EXECUTING|BUILT|COMPLETED)
       - Required fields: version, goal, state, scope, current_phase
       - Valid values: State enum, ParallelMode (in-repo|worktree), ColonyScope (project|meta), PhaseStatus, TaskStatus
       - Consistency rules: EXECUTING requires current_phase > 0, paused flag only valid with READY state
       - Common failures: missing goal, invalid state transition, orphaned worktrees, deprecated signals array
       - Remedies: normalize legacy states, remove orphaned worktrees, migrate signals to pheromones.json

       **## pheromones.json Healthy State**
       - Schema: PheromoneSignal fields (id, type, priority, source, content, active, expires_at, content_hash)
       - Required fields: id, type, active, created_at
       - Valid values: type in (FOCUS|REDIRECT|FEEDBACK), content as valid json.RawMessage
       - Common failures: expired-but-active signals, missing IDs, invalid types, duplicate content hashes
       - Remedies: deactivate expired signals, generate missing IDs, default invalid types to FOCUS

       **## session.json Healthy State**
       - Schema: SessionFile fields
       - Required fields: session_id, started_at, colony_goal
       - Cross-reference: current_phase and colony_goal must match COLONY_STATE
       - Staleness: >7 days = warning, >30 days = critical
       - Common failures: phase/goal mismatch with state, stale sessions, missing session_id
       - Remedies: sync phase and goal from COLONY_STATE

       **## Data Files Healthy State**
       - midden/midden.json: valid JSON, acknowledged entries optional
       - instincts.json: valid JSON, each entry has trigger and action
       - learning-observations.json: valid JSON, content_hash present
       - assumptions.json: valid JSON, confidence in (confident|likely|unclear)
       - pending-decisions.json: valid JSON, type in (blocker|issue|note)
       - constraints.json: should be empty `{}` (ghost file — Go code ignores content)
       - Common failures: corrupted JSON, empty required files
       - Remedies: JSON recovery with --force, reset constraints to `{}`

       **## JSONL Files Healthy State**
       - trace.jsonl: each line valid JSON with id, timestamp, level, topic
       - event-bus.jsonl: each line valid JSON with id, topic, timestamp, ttl_days
       - Rotation: trace.jsonl at 50MB
       - Common failures: malformed lines, expired events, approaching rotation limit
       - Remedies: malformed lines are auto-skipped by storage layer

       **## Wrapper Parity (Deep Scan)**
       - Expected counts: 50 commands (YAML/Claude/OpenCode), 25 agents (Claude/OpenCode/Codex/mirrors), 10+18 skills
       - Common failures: count mismatch across surfaces, missing files after update
       - Remedies: manual — run `aether install` to regenerate wrappers

       **## Version Compatibility**
       - COLONY_STATE version "3.0" is current
       - Legacy states (PAUSED, PLANNED, SEALED, ENTOMBED) auto-normalized on load
       - No structural migration framework — only state name normalization
       - Cache files (.cache_*) are expected and auto-rebuilt
