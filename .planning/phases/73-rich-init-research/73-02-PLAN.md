    **Governance Deep Parsing (INIT-05, D-05, D-06):**

    **1. Add `governanceDetail` struct:**
    ```go
    type governanceDetail struct {
        Tool     string                 `json:"tool"`
        File     string                 `json:"file"`
        Category string                 `json:"category"`
        Rules    map[string]interface{} `json:"rules,omitempty"`
        Extends  []string               `json:"extends,omitempty"`
        Config   map[string]interface{} `json:"config,omitempty"`
    }
    ```

    **2. Implement deep governance parser functions.** For each governance file detected by the existing `governanceDetectors`, create a parser that extracts actual content. Key parsers:

    **parseEslintrcDeep(target string) *governanceDetail:**
    - Try `.eslintrc.json` first: use `gjson.GetBytes` for `rules` (iterate map) and `extends` (iterate array)
    - Try `.eslintrc.yml`: use `yaml.Unmarshal` for same fields
    - Try `.eslintrc` (plain JSON or YAML): try JSON first, fallback to YAML
    - Return `governanceDetail{Tool: "ESLint", File: name, Category: "linter", Rules: ..., Extends: ...}`

    **parseGolangciDeep(target string) *governanceDetail:**
    - Try `.golangci.yml`, `.golangci.yaml`, `golangci.yml`
    - Use `yaml.Unmarshal` into `map[string]interface{}`
    - Extract `linters.enable` array, `issues.exclude-rules` array
    - Put enabled linters in `Config["enabled_linters"]`, exclude rules count in `Config["exclude_rules_count"]`
    - Return `governanceDetail{Tool: "golangci-lint", File: name, Category: "linter", Config: ...}`

    **parsePrettierDeep(target string) *governanceDetail:**
    - Try `.prettierrc`, `.prettierrc.json`
    - Use `gjson.GetBytes` to extract top-level config keys (semi, singleQuote, tabWidth, printWidth, trailingComma, etc.)
    - Return `governanceDetail{Tool: "Prettier", File: name, Category: "formatter", Config: ...}`

    **parseBiomeDeep(target string) *governanceDetail:**
    - Read `biome.json`
    - Use `gjson.GetBytes` for `formatter` and `linter` sections
    - Return `governanceDetail{Tool: "Biome", File: "biome.json", Category: "formatter", Config: ...}`

    **parseJestDeep(target string) *governanceDetail:**
    - Read `jest.config.js` or `jest.config.ts`
    - Use regex to find `testMatch`, `preset`, `testEnvironment`, `transform` assignments
    - Return `governanceDetail{Tool: "Jest", File: name, Category: "test", Config: ...}`

    **parseVitestDeep(target string) *governanceDetail:**
    - Read `vitest.config.ts` or `vitest.config.js`
    - Use regex to find `include`, `exclude`, `coverage` patterns
    - Return `governanceDetail{Tool: "Vitest", File: name, Category: "test", Config: ...}`

    **parseGHActionsDeep(target string) []governanceDetail:**
    - Glob `.github/workflows/*.yml` and `.github/workflows/*.yaml`
    - For each file, parse with yaml.Unmarshal
    - Extract `name` (workflow name), `on` (triggers), count of `jobs` entries, count of steps across all jobs
    - Return one `governanceDetail` per workflow file with `{Category: "ci", Config: {name, triggers, job_count, step_count}}`

    **parseGitlabCIDeep(target string) *governanceDetail:**
    - Read `.gitlab-ci.yml`
    - Parse with yaml.Unmarshal
    - Extract `stages` array, count of top-level job keys (excluding `stages`, `variables`, `default`, `include`)
    - Return `governanceDetail{Tool: "GitLab CI", File: ".gitlab-ci.yml", Category: "ci", Config: ...}`

    **parseMakefileDeep(target string) *governanceDetail:**
    - Read `Makefile`
    - Use regex `^([a-zA-Z0-9_-]+):` to find targets
    - Return `governanceDetail{Tool: "Make", File: "Makefile", Category: "build", Config: {"targets": [...]}}`

    **parseTaskfileDeep(target string) *governanceDetail:**
    - Read `Taskfile.yml`
    - Parse with yaml.Unmarshal
    - Extract `tasks` map keys (task names)
    - Return `governanceDetail{Tool: "Task", File: "Taskfile.yml", Category: "build", Config: {"tasks": [...]}}`

    **parseJustfileDeep(target string) *governanceDetail:**
    - Read `justfile`
    - Use regex `^([a-zA-Z0-9_-]+)` to find recipes (lines not starting with space/tab that end with colon or have a recipe line)
    - Return `governanceDetail{Tool: "Just", File: "justfile", Category: "build", Config: {"recipes": [...]}}`

    **3. Add orchestrating function `deepParseGovernance`:**
    ```go
    func deepParseGovernance(target string) []governanceDetail {
        var details []governanceDetail

        if d := parseEslintrcDeep(target); d != nil { details = append(details, *d) }
        if d := parseGolangciDeep(target); d != nil { details = append(details, *d) }
        if d := parsePrettierDeep(target); d != nil { details = append(details, *d) }
        if d := parseBiomeDeep(target); d != nil { details = append(details, *d) }
        if d := parseJestDeep(target); d != nil { details = append(details, *d) }
        if d := parseVitestDeep(target); d != nil { details = append(details, *d) }
        if ds := parseGHActionsDeep(target); len(ds) > 0 { details = append(details, ds...) }
        if d := parseGitlabCIDeep(target); d != nil { details = append(details, *d) }
        if d := parseMakefileDeep(target); d != nil { details = append(details, *d) }
        if d := parseTaskfileDeep(target); d != nil { details = append(details, *d) }
        if d := parseJustfileDeep(target); d != nil { details = append(details, *d) }

        return details
    }
    ```
