<objective>
Add --verbose flag to aether build command and implement output filtering so that raw worker agent output is suppressed by default (per D-01, D-02, D-03, D-04).

Purpose: Build output is currently unfiltered -- users see everything workers produce. This makes builds noisy and hard to scan. The --verbose flag lets users opt into full output for debugging while keeping the default experience clean.

Output: New output_filter.go with filtering functions, --verbose flag on buildCmd, Verbose field on codexBuildOptions, RawOutput printing gated by verbose flag, comprehensive tests.
</objective>

    Create `cmd/output_filter.go` in package `cmd`:

    1. Define a package-level `var buildVerbose bool` that controls filtering.
    2. Create `func setBuildVerbose(v bool)` to set the flag.
    3. Create `func filteredPrintln(a ...interface{})` that calls `fmt.Fprintln(stdout, a...)` only when `buildVerbose == true`. When false, silently discards.
    4. Create `func filteredFprintf(format string, a ...interface{})` that calls `fmt.Fprintf(stdout, format, a...)` only when `buildVerbose == true`. When false, silently discards.
    5. Create `func filteredFprintln(w io.Writer, a ...interface{})` that writes only when `buildVerbose == true`.

    These functions target ONLY raw worker output (the `WorkerResult.RawOutput` field).     - Test `setBuildVerbose(true)` then `filteredPrintln("test")` -- assert stdoutBuf contains "test"
    - Test `setBuildVerbose(false)` then `filteredPrintln("test")` -- assert stdoutBuf is empty
    - Test `filteredFprintf("hello %s", "world")` in both modes
    - Reset `buildVerbose = false` in each test and restore `stdout = os.Stdout` in defer
  </action>
