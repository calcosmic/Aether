<objective>
Fix the internal continue watcher path to treat timeout as advisory when runtime verification passes, mirroring the Phase 64.1 fix for the external path.

Purpose: The internal watcher path (`runCodexContinueVerification`) hard-blocks on any watcher failure including timeout. This is the primary cause of the "stuck in continue loops" bug. When the watcher agent times out but build/types/lint/tests all passed independently, the timeout should be advisory (warning) not blocking.

Output: Modified `cmd/codex_continue.go` with advisory timeout pattern and new tests in `cmd/codex_continue_test.go`.
</objective>
