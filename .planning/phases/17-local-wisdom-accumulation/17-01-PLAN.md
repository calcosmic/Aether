    **1. Add `_queen_write_learnings()` function to queen.sh:**

    New function that directly appends build learnings to the `## Build Learnings` section of QUEEN.md. This is a NEW write path that bypasses observation thresholds (per user decision: every build writes learnings).

    Signature: `queen-write-learnings <phase_id> <phase_name> <learnings_json>`
    - `learnings_json` is a JSON array: `[{"claim":"what happened","tag":"repo|general","evidence":"..."}]`

    Implementation (follow existing queen-promote patterns):
    1. Validate args (phase_id, phase_name, learnings_json all required)
    2. Find QUEEN.md at `$AETHER_ROOT/.aether/QUEEN.md`, error if missing
    3. Get timestamp and date
    4. Iterate learnings_json entries via `jq -r '.[] | @base64'` loop
    5. For each entry: extract claim, tag (default "repo"), evidence
    6. Dedup check: `grep -Fq -- "$claim" "$queen_file"` -- skip if already present
    7. Build entry: `- [$tag] $claim -- *Phase $phase_id ($phase_name)* ($date_short)`
    8. Group entries under a phase subsection header: `### Phase $phase_id: $phase_name`
       - If this subsection already exists, append entries after it
       - If not, create it at the end of the Build Learnings section (before the next `---` or `## ` header)
    9. Replace placeholder text `*No build learnings recorded yet.*` if present
    10. Use temp file + mv for atomic write (same pattern as queen-promote)
    11. Update Evolution Log with a single entry for the batch
    12. Update METADATA stats: increment `total_build_learnings` by count
    13. Return: `json_ok '{"written":N,"phase":"P","timestamp":"T"}'`

    **2. Add `_queen_promote_instinct()` function to queen.sh:**

    New function that writes a high-confidence instinct to the `## Instincts` section.

    Signature: `queen-promote-instinct <trigger> <action> <confidence> <domain>`

    Implementation:
    1. Validate args (trigger and action required, confidence defaults to 0.8, domain defaults to "workflow")
    2. Find QUEEN.md, error if missing
    3. Dedup check: `grep -Fq -- "$action" "$queen_file"` -- skip if already present, return `json_ok '{"written":0,"reason":"duplicate"}'`
    4. Build entry: `- [instinct] **$domain** ($confidence): When $trigger, then $action`
    5. Insert into `## Instincts` section (replace placeholder if present, otherwise append)
    6. Temp file + mv atomic write
    7. Update Evolution Log
    8. Update METADATA stats: increment `total_instincts`
    9. Return: `json_ok '{"promoted":true,"domain":"D","confidence":C}'`
