# Plan 29-01: Trace Diagnostics

## Goal
Build `aether medic --trace <file>` that accepts a trace export JSON and produces
a standalone diagnostic summary — no access to the user's repo needed.

## Tasks

### Task 1: Trace diagnostic analyzer (`cmd/medic_trace.go`)
New file with:
- `TraceDiagnostic` struct: run_id, entry_count, duration, timeline, error_clusters,
  state_gaps, suggestions, health_issues
- `analyzeTraceDiagnostics(entries []TraceEntry) TraceDiagnostic`:
  - Reconstruct state transition timeline
  - Identify error clusters (3+ errors in same phase = cluster)
  - Detect state gaps (missing transitions like building→active without planning)
  - Calculate token totals and cost
  - Detect stalled phases (phase started but no subsequent activity for >30min)
  - Generate fix suggestions per error cluster
- `renderTraceDiagnostic(diag TraceDiagnostic) string`:
  - Human-readable report with stage markers (Summary, Timeline, Errors, Suggestions)
- `renderTraceDiagnosticJSON(diag TraceDiagnostic) string`

### Task 2: Wire into medic command
- Add `--trace <file>` flag to medic command
- When `--trace` is provided, load trace export JSON, run analyzer, output report
- Skip normal colony health scan when trace mode is active
- Does NOT require an active colony (works from export alone)

### Task 3: Tests (`cmd/medic_trace_test.go`)
- Test error cluster detection (3+ errors in same phase)
- Test state gap detection (missing transitions)
- Test stalled phase detection
- Test suggestion generation
- Test rendering (both text and JSON)
- Test with empty trace
- Test with trace from file (temp file)

## Success Criteria (from R043)
1. Accepts trace export JSON and produces diagnostic summary
2. Reconstructs state transition timeline
3. Identifies error clusters and intervention points
4. Suggests fixes based on trace patterns
5. Works without access to user's repo
