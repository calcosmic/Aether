  <name>Task 1: Add promote_to_colony function to oracle.sh and extend validate-oracle-state</name>
  <files>.aether/oracle/oracle.sh, .aether/aether-utils.sh</files>
  <action>
Add a `promote_to_colony` function to oracle.sh. Place it after the existing `run_synthesis_pass` function (before the main loop). Follow the design from 11-RESEARCH.md Pattern 1:

1. **promote_to_colony function** (oracle.sh):
   - Takes 3 args: plan_file, state_file, aether_root
   - Derives utils path: `$aether_root/.aether/aether-utils.sh`
   - Reads status from state.json -- reject with error if status is "active" (only allow "complete" or "stopped")
   - Check COLONY_STATE.json exists at `$aether_root/.aether/data/COLONY_STATE.json` -- reject if missing
   - Read topic from state.json
   - Extract high-confidence answered questions: `jq -c '[.questions[] | select(.status == "answered" and .confidence >= 80)]' "$plan_file"`
   - If count is 0, print "No findings meet promotion threshold" and return 0
   - Loop through qualifying questions. For each:
     - Extract q_text and q_confidence via jq
     - Call `bash "$utils" instinct-create --trigger "When researching: $q_text" --action "Oracle found (${q_confidence}% confidence): $(findings_text)" --confidence "$(echo "scale=2; $q_confidence / 100" | bc)" --domain "research" --source "oracle:$topic" --evidence "Oracle research: $q_text" 2>/dev/null || true`
     - Call `bash "$utils" learning-promote "Oracle: $q_text -- $(first_finding)" "oracle" "oracle-research" "oracle,research" 2>/dev/null || true`
     - Call `bash "$utils" memory-capture learning "Oracle research finding: $q_text (${q_confidence}%)" "pattern" "oracle:promote" 2>/dev/null || true`
   - Truncate finding text to 200 chars for instinct action field using `head -c 200`
   - Handle both v1.1 structured findings `{text, source_ids, iteration}` and v1.0 string findings using jq: `[.key_findings[].text // .key_findings[]] | join("; ")`
   - Print summary of promoted count at the end

2. **validate-oracle-state template field** (aether-utils.sh):
   - In the validate-oracle-state `state)` case, after the existing strategy and focus_areas checks, add optional template field validation:
   - Pattern: `if has("template") then enum("template";["tech-eval","architecture-review","bug-investigation","best-practices","custom"]) else "pass" end`
   - This follows the exact same `if has(...) then validate else pass` pattern used for strategy and focus_areas in Phase 10
   - The template field is optional -- state.json files without it remain valid (backward compatibility)

Important: The promote_to_colony while-loop subshell variable issue -- use a process substitution or temp file pattern instead of piping into while-read if you need the promoted counter to survive the loop. The research code shows a subshell pipe pattern that would lose the counter; fix this by using `< <(...)` or writing count to a temp file.
  </action>
