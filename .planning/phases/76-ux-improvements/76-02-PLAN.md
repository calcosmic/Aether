  **Part A: Warning Computation Functions**

  Add the following new functions to `cmd/status.go`:

  - Function `computeWarnings(state colony.ColonyState, s *storage.Store) []string`:
    Returns a list of warning strings. Empty list means no warnings. Each warning is a single line.

    1. **Stale state warning (per D-12):**
       - If `state.InitializedAt` is not nil AND `time.Since(*state.InitializedAt) > 7*24*time.Hour`, add:
         `Stale: colony was last active more than 7 days ago. Recent work may not be reflected.`
       - Also check the most recent phase timestamp if available. Look at `state.Plan.GeneratedAt` -- if it exists and `time.Since(*state.Plan.GeneratedAt) > 7*24*time.Hour`, add the stale warning.

    2. **Failed phases warning (per D-12):**
       - Iterate `state.Plan.Phases`. For each phase where `phase.Status == colony.PhaseFailed`:
         Add: `Failed phase %d (%s). Run \`aether build %d\` to retry.` (using phase.ID and phase.Name)

    3. **Unacknowledged midden warning (per D-12):**
       - Load midden file: `var mf colony.MiddenFile; s.LoadJSON("midden.json", &mf)` (ignore error -- file may not exist)
       - Count entries where `entry.Acknowledged == nil || *entry.Acknowledged == false`
       - If count > 0, add: `%d unacknowledged failure(s). Run \`aether midden-review\` to inspect.`

    4. **Pheromone expiry warning (per D-12):**
       - Load pheromone file: `var pf colony.PheromoneFile; s.LoadJSON("pheromones.json", &pf)` (ignore error)
       - For each active signal where `ExpiresAt != nil`:
         Parse the expiry time. If `time.Until(expiry) < 3*24*time.Hour` (less than 3 days remaining), add:
         `Expiring signal: %s -- %s (expires in less than 3 days)` (using sig.Type and extractContentText(sig.Content) truncated to 40 chars)
       - Skip this check for signals without ExpiresAt (REDIRECT signals that persist indefinitely)

  - Function `renderWarningsSection(warnings []string) string`:
    1. If `len(warnings) == 0`, return empty string
    2. Build string:
       - `renderBanner("\u26A0\uFE0F", "Warnings")` (warning sign emoji -- actual Unicode character, NOT the string "warning-sign")
       - `visualDivider`
       - For each warning: `warning + "\n"`
       - Blank line after all warnings
    3. Return the built string

  CRITICAL: `renderBanner` takes two string parameters and renders them directly as `━━ {emoji} {title} ━━`. The first parameter MUST be an actual emoji character, NOT a descriptive string name like `"warning-sign"`. The codebase uses `commandEmoji("status")` which returns `"📊"` (actual emoji). The warnings banner MUST use the actual warning sign emoji character `"\u26A0\uFE0F"` (or the literal `"⚠️"`).
