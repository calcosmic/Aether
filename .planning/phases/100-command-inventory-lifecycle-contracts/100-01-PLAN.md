Per D-01, D-03, D-08, D-09: Create cmd/audit_catalog.go with a new audit-catalog Cobra subcommand.

1. Define CatalogEntry struct with JSON tags exactly per D-03:
   - name (string)
   - short_description (string)
   - flags ([]string) -- extracted via cmd.LocalFlags().VisitAll()
   - parent_command (string) -- empty string for root-level commands
   - has_subcommands (bool) -- len(cmd.Commands()) > 0
   - output_mode (string) -- default "unknown"; classify "json" if RunE body uses outputOK, "visual" if outputWorkflow. Accept that many will be "unknown" per research Pitfall 3.

2. Implement buildAuditCatalog(root *cobra.Command) []CatalogEntry that calls walkCommands recursively. The walkCommands function:
   - Iterates cmd.Commands() (which returns sorted children per Cobra API)
   - Skips commands where cmd.IsAvailableCommand() returns false (filters help/completion per research Pitfall 2)
   - For each available command: extract flags via LocalFlags().VisitAll(), build CatalogEntry, recurse into children
   - Parent name is passed down through recursion

3. Define auditCatalogCmd as &cobra.Command with:
   - Use: "audit-catalog"
   - Short: "Produce structured catalog of all registered commands"
   - Args: cobra.NoArgs
   - --json flag (bool, default false)
   - RunE: calls buildAuditCatalog(rootCmd), then either outputs JSON via fmt.Fprintln(stdout, ...) when --json is true, or calls outputWorkflow(catalog, renderAuditCatalogVisual(catalog)) for visual mode
   - Follow the exact pattern from source_check.go (referenced in interfaces above)

4. Implement renderAuditCatalogVisual(catalog []CatalogEntry) string:
   - Use renderBanner("catalog", "Command Catalog") as header
   - Table with columns: Name, Flags, Output Mode, Subcommands
   - Summary line: "Total: N commands"

5. Register via init() function: rootCmd.AddCommand(auditCatalogCmd)

6. Add "audit-catalog" to skipStoreInit() in cmd/root.go -- this command reads the Cobra tree, not the store.

7. Create cmd/audit_catalog_test.go with three test functions:

   a. TestAuditCatalogGolden:
      - Call buildAuditCatalog(rootCmd) directly (per D-09, no subprocess)
      - Marshal to JSON with json.MarshalIndent(catalog, "", "  ")
      - Support -update-golden flag: if set, write to cmd/testdata/command_catalog.json
      - Otherwise read golden file and compare with bytes.Equal
      - On mismatch, fail with message: "catalog golden mismatch; run with -update-golden to refresh"
      - Define updateGolden flag: var updateGolden = flag.Bool("update-golden", false, "update golden files")

   b. TestCatalogCompleteness:
      - Build catalog, verify all 16 lifecycle commands from D-06 are present in catalog entries (by name)
      - Expected names: init, discuss, colonize, plan, build, continue, seal, entomb, publish, update, recover, status, resume, watch, patrol, profile
      - Assert count of catalog entries is >= 300 (current expected ~317+ per research)
      - Fail with missing names listed if any are absent

   c. TestCatalogSchema:
      - Build catalog, iterate every entry
      - Assert each entry has non-empty name and short_description
      - Assert flags field is present (can be empty [] for commands with no flags)
      - Assert parent_command is a string (empty for root-level)
      - Assert has_subcommands is a bool
      - Assert output_mode is one of: "json", "visual", "json+visual", "text", "unknown"

8. After tests pass, generate the initial golden file by running:
