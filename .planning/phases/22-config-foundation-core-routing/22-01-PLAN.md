  <name>Task 3: Deprecate spawn-with-model.sh with backward-compatible warning</name>
  <files>.aether/utils/spawn-with-model.sh</files>
  <action>
Add a deprecation warning to `.aether/utils/spawn-with-model.sh` as the first functional action (after argument validation, before model lookup). The script is superseded by agent frontmatter `model:` fields which handle routing natively through Claude Code.

**Add this block after the argument validation block (line 20) and before the model lookup (line 23):**
```bash
# DEPRECATED: Per-caste model routing is now handled by agent frontmatter `model:` field.
# This script is preserved for backward compatibility but will be removed in a future version.
# See: .aether/workers.md "Model Selection" section for current routing approach.
echo "[DEPRECATED] spawn-with-model.sh is superseded by agent frontmatter model: field" >&2
echo "  Per-caste routing is now handled by Claude Code natively." >&2
echo "  This script will be removed in a future version." >&2
```

**Also update the fallback model** on line 23-24. The current fallback is `glm-5-turbo`. After the YAML change, `model-profile get` will return slot names (e.g., `sonnet`). To maintain backward compatibility, add slot resolution using the new `model_slots` section:

Replace:
```bash
model=$(echo "$model_info" | jq -r '.result.model // "glm-5-turbo"')
```
With:
```bash
slot=$(echo "$model_info" | jq -r '.result.model // "inherit"')
# Resolve slot name to concrete model via model_slots in YAML
case "$slot" in
  opus|sonnet|haiku)
    # Read from model_slots section in YAML
    resolved=$(node -e "const p=require('./bin/lib/model-profiles').loadModelProfiles('.'); console.log(p.model_slots?.['$slot']||'glm-5-turbo')" 2>/dev/null)
    model="${resolved:-glm-5-turbo}"
    ;;
  inherit|*)
    model="glm-5-turbo"
    ;;
esac
```

This ensures spawn-with-model.sh still sets a concrete model name in `ANTHROPIC_MODEL`, even though the YAML now stores slot names.
