  <name>Task 2: Version bump and pre-publish checks</name>
  <files>package.json, .aether/version.json</files>
  <action>
Per user decision: go straight to v2.1.0 (no RC). Per research recommendation: leave push to remote for user.

1. Ensure git working tree is clean. All changes from Plan 01 and any walkthrough fixes must be committed. If there are uncommitted changes, commit them first with an appropriate message.

2. Update `.aether/version.json` to version `2.1.0` with current timestamp. This file tracks the internal Aether version.

3. Run `npm version 2.1.0 -m "chore(release): bump version to v%s"`. This atomically:
   - Updates `package.json` version to 2.1.0
   - Creates a git commit with the message
   - Creates git tag `v2.1.0`
   - Runs `prepublishOnly` which calls `validate-package.sh` (must pass)

4. Verify the tag exists: `git tag --list 'v2.1*'`

5. Run a final `npm publish --dry-run` to show exactly what would be published. Display the output for review.
  </action>
  <verify>Run `node -e "console.log(require('./package.json').version)"` -- should print `2.1.0`. Run `git tag --list 'v2.1*'` -- should show `v2.1.0`. Run `npm publish --dry-run` -- should succeed with no errors.</verify>
  <done>package.json version is 2.1.0. Git tag v2.1.0 exists. version.json updated. npm publish --dry-run succeeds.</done>
