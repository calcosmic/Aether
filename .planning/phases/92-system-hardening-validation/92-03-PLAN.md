- Verify dispatches are created, FakeInvoker results are returned

**Step 3: Trigger gate failure**
- Write `gate-results.json` with a failed gate (e.g., "tests_gate": "failed")
- Verify gate state is persisted

**Step 4: Unblock**
- Simulate unblock flow by reading gate-results.json, verifying it shows the failed gate
- Write updated gate-results.json with the gate now passing (simulating fixer success)

**Step 5: Fixer dispatch**
- Create a fixer dispatch, invoke with FakeInvoker
- Verify fixer result is structured correctly (status, files_modified, etc.)

**Step 6: Continue with verification**
- Call the continue flow (or simulate its steps)
- Verify all gates pass
- Verify phase advances from 1 to 2

**Step 7: Learning capture**
- Write a learning entry to the ColonyStore
- Verify the entry has proper evidence (run_id, worker, files, confidence)
- Verify classification is correct

**Step 8: Hive search**
- Initialize SQLite colony.db in the temp directory
- Insert test data for workers and gates
- Call hive search and verify FTS5 query returns results

**Step 9: Skill lifecycle**

- Verify skill can be listed (index shows it)
- Verify skill has proper frontmatter (evidence, confidence)

- Set colony state to CROWNED_ANTHILL
- Verify seal flow cleans up temporary data
- Verify heartbeat files are removed

**Step 11: Process cleanup**
- Verify no orphaned worker processes remain in worker-processes.json
- Verify cleanup functions run without error

Each step uses `t.Fatalf` on critical failures and `t.Errorf` for non-critical assertions. The test is self-contained -- all state is in the temp directory.
