
   ```

     research_content="${research_content:0:$research_budget}"
     echo "[research] trimmed to ${research_budget} chars" >&2
   fi
   ```
   This follows the same pattern as `_skill_inject` in skills.sh (simple char counting with truncation logging).

4. **Wrap in injection block:**
   ```
   research_context="--- PHASE RESEARCH (Domain Knowledge) ---
   ${research_content}
   --- END PHASE RESEARCH ---"
   ```

   ```
   Research loaded: phase-{phase_id}-research.md ({research_word_count} words)
   ```

6. **Store for worker injection:** The `research_context` variable is now available for build-wave.md and build-verify.md to inject into worker prompts.
