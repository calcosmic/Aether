    Per D-05 (spawn workers as subprocess exec calls per manifest), D-06 (real spawn-log/spawn-complete via Go CLI):

    1. Create `.aether/ts-host/src/worker-dispatch.ts`:

       Export a `DispatchResult` interface: `{ name: string; status: TerminalStatus; summary: string; duration?: number }`

       Export a `DispatchOptions` interface extending `GoBridgeOptions`: `{ simulateWorkers?: boolean }` (default true for prototype)

       Export async function `dispatchSingleWorker(opts: DispatchOptions, dispatch: BuildDispatch): Promise<DispatchResult>`:
       - Before dispatch: call `callGoJSON(opts, ["spawn-log", "--parent", "Queen", "--caste", dispatch.caste, "--name", dispatch.name, "--task", dispatch.task, "--depth", "1"])`. Assert result.recorded === true. If error, log warning but continue (spawn-log failure should not block dispatch).
       - If opts.simulateWorkers is true (default): simulate worker execution with `await new Promise(r => setTimeout(r, 100))` (brief delay to simulate work), return `{ name: dispatch.name, status: "completed", summary: "Simulated worker completion for " + dispatch.name, duration: 0.1 }`
       - If simulateWorkers is false: spawn actual platform worker using `execFileSync` or `execFile` with the platform CLI. For the prototype, this path can throw "Real worker dispatch not yet implemented".
       - After dispatch: call `callGoJSON(opts, ["spawn-complete", "--name", dispatch.name, "--status", result.status, "--summary", result.summary])`. Assert result.completed === true.
       - Return the DispatchResult.
       - Wrap the entire dispatch in try/catch: on any error, still attempt spawn-complete with status "failed" and the error message as summary, then re-throw.

       Export async function `dispatchWorkers(opts: DispatchOptions, dispatches: BuildDispatch[]): Promise<DispatchResult[]>`:
       - Group dispatches by wave number (dispatch.wave, default to 0 if missing).
       - Process waves sequentially: for each wave, dispatch all workers in that wave.
       - Within a wave, dispatch workers sequentially (parallel dispatch within a wave can be added later).
       - Collect all DispatchResult objects and return them.
       - Log progress to stderr: "Dispatching wave N: X workers" before each wave, "Wave N complete" after.

       Export function `toWorkerResults(dispatches: BuildDispatch[], results: DispatchResult[]): WorkerResult[]`:
       - Map each dispatch + its corresponding result to a WorkerResult object, preserving name, status, summary from the result, and caste, task_id, stage, wave from the dispatch.

       - Test: "spawn-complete records after dispatch" -- same setup, verify spawn-complete was called by checking the spawn tree entry has status "completed" (not "spawned")
       - Test: "dispatchWorkers processes multiple dispatches" -- call dispatchWorkers with 2+ dispatches from a manifest, verify all return completed status, verify spawn tree has entries for all dispatches
       - Test: "failed worker records spawn-complete with failed status" -- create a mock scenario where the dispatch fails (temporarily make callGoJSON fail), verify spawn-complete is still called with status "failed"
       - Test: "toWorkerResults maps dispatches to WorkerResult array" -- create test dispatches and results, call toWorkerResults, verify output has correct fields

       Test setup: Use same `setupTestColony(t)` helper pattern from Plan 01. Call `aether build 1 --plan-only` to get a manifest with real dispatches, then use those dispatches in tests.
