## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TS host -> Go binary | TS host invokes Go CLI with user-influenced args; Go validates all inputs |
| TS host -> filesystem | TS host writes completion files to tmpdir only; Go-owned paths protected |
| TS host -> platform workers | Worker dispatch uses manifest data from Go (trusted), not user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-109-01 | Tampering | go-bridge.ts callGoJSON | mitigate | Use execFileSync (no shell interpolation), Go validates all string fields |
| T-109-02 | Tampering | writeCompletionFile | mitigate | assertNoDirectDataWrites enforces GO_OWNED_PATHS; writes only to os.tmpdir() |
| T-109-03 | Denial of Service | go-bridge.ts JSON parse | mitigate | try/catch around JSON.parse and execFileSync; maxBuffer limit of 10MB |
| T-109-04 | Spoofing | Manifest staleness | accept | Go finalizer validates generated_at freshness (24h max); TS host does not need to re-validate |
</threat_model>
