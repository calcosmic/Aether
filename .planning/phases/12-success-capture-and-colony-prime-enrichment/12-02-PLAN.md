The block must follow the established pattern used by the blocker injection and decision injection sections in colony-prime (conditional section with `cp_final_prompt+=` concatenation and `cp_log_line` augmentation).

Insert this exact code:

```bash
    # === Rolling-summary injection (MEM-02) ===
    # Read last 5 entries directly (not via context-capsule which truncates)
    cp_roll_count=5
    cp_roll_entries=""
    if [[ -f "$DATA_DIR/rolling-summary.log" ]]; then
      cp_roll_entries=$(tail -n "$cp_roll_count" "$DATA_DIR/rolling-summary.log" 2>/dev/null | \
        awk -F'|' 'NF >= 4 {printf "- [%s] %s: %s\n", $1, $2, $4}')
    fi

    if [[ -n "$cp_roll_entries" ]]; then
      cp_final_prompt+=$'\n'"--- RECENT ACTIVITY (Colony Narrative) ---"$'\n'
      cp_final_prompt+="$cp_roll_entries"$'\n'
      cp_final_prompt+="--- END RECENT ACTIVITY ---"$'\n'

      cp_roll_actual=$(echo "$cp_roll_entries" | grep -c '.' || echo "0")
      cp_log_line="$cp_log_line, $cp_roll_actual activity entries"
    fi
    # === END rolling-summary injection ===
```

Key design decisions:
- Uses `tail -n 5` to read last 5 entries from the pipe-delimited log file
- Formats each entry as `- [timestamp] event_type: summary` (fields 1, 2, 4 from the pipe-delimited format, skipping field 3 which is source)
- Section header follows existing convention: `--- SECTION NAME ---` and `--- END SECTION NAME ---`
- The section is NOT inside context-capsule, so it cannot be truncated by word limits
- When rolling-summary.log is missing or empty, no section is added (graceful degradation)
- `cp_log_line` is augmented with the activity count for the colony-prime log output
- Accept minor duplication with context-capsule's 3 entries (research recommendation: option b)

Do NOT modify any other part of colony-prime. Do NOT modify context-capsule.
