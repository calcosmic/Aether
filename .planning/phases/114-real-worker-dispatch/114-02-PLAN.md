    Create `.aether/ts-host/src/wave-orchestrator.ts` with:

    1. `export interface WaveOrchestratorOptions extends DispatchOptions`
       - Add `parallel?: boolean` (default: true)
       - Add `retryLimit?: number` (default: 2)
       - Add `retryDelayMs?: number` (default: 5000)
       - Add `timeoutMs?: number` (default: 600000 = 10 min)
    2. `export interface WaveResult {
         wave: number;
         results: DispatchResult[];
         failures: DispatchResult[];
         retried: number;
       }`
    3. `export async function dispatchWave(
         opts: WaveOrchestratorOptions,
         dispatches: BuildDispatch[]
       ): Promise<WaveResult>`
       - If `opts.parallel` and dispatches.length > 1:
         - Use `Promise.all(dispatches.map(d => dispatchSingleWorker(opts, d)))`
       - Else:
         - Dispatch sequentially: `for (const d of dispatches) { results.push(await dispatchSingleWorker(opts, d)); }`
       - Return `{ wave, results, failures, retried }`
    4. `export async function retryDispatch(
         opts: WaveOrchestratorOptions,
         dispatch: BuildDispatch,
         attempt: number
       ): Promise<DispatchResult>`
       - Call `dispatchSingleWorker(opts, dispatch)`
       - If result.status === "failed" and attempt < (opts.retryLimit ?? 2):
         - Wait `opts.retryDelayMs * attempt` ms (exponential backoff)
         - Log retry attempt to stderr
         - Recursively call `retryDispatch(opts, dispatch, attempt + 1)`
       - Return result
    5. `export async function dispatchWaves(
         opts: WaveOrchestratorOptions,
         dispatches: BuildDispatch[]
       ): Promise<WaveResult[]>`
       - Group dispatches by wave (same logic as current dispatchWorkers)
       - For each wave (sorted ascending):
         - Emit ceremony event (optional — can be stub)
         - Call `dispatchWave(opts, waveDispatches)`
         - Collect results
       - Return array of WaveResult

    Use `node:child_process` for spawn (already in platform-dispatcher).
    Emit ceremony events via `process.stderr.write` for now (narrator will pick up real events in Phase 115).
  </action>
