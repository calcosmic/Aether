Tests to implement (8 tests total):

**QUEEN-01 Tests (auto-promotion during continue):**

1. "learning-promote-auto promotes observation meeting auto threshold (pattern type)"
   - Setup: Create colony, call learning-observe TWICE for same pattern content from different colonies (auto threshold for pattern = 2)
   - Act: Call learning-promote-auto with the content
   - Assert: result.promoted === true, QUEEN.md contains the content in Patterns section

2. "learning-promote-auto skips observation below auto threshold"
   - Setup: Create colony, call learning-observe ONCE for pattern content (auto threshold = 2)
   - Act: Call learning-promote-auto
   - Assert: result.promoted === false, result.reason === "threshold_not_met"

3. "learning-promote-auto prevents double-promotion (idempotency guard)"
   - Setup: Create colony, observe pattern content twice, call learning-promote-auto (should promote)
   - Act: Call learning-promote-auto AGAIN with the same content
   - Assert: Second call returns promoted === false with reason === "already_promoted", QUEEN.md contains the content exactly ONCE (not duplicated)

4. "memory-capture triggers auto-promotion on recurrence (end-to-end continue path)"
   - Setup: Create colony with pheromones.json
   - Act: Call memory-capture twice for same content with type "learning"
   - Assert: Second call returns auto_promoted === true, QUEEN.md contains the promoted content

**QUEEN-02 Tests (batch promotion during seal):**

5. "batch sweep promotes multiple observations meeting different thresholds"
   - Setup: Create colony, observe pattern content 2x (meets pattern auto=2), observe philosophy content 3x (meets philosophy auto=3), observe another pattern only 1x (below threshold)
   - Act: For each observation in learning-observations.json, call learning-promote-auto
   - Assert: Pattern and philosophy content appear in QUEEN.md. Below-threshold content does NOT appear in QUEEN.md.

6. "batch sweep is idempotent (safe to run after memory-capture already promoted)"
   - Setup: Create colony, observe pattern 2x, call memory-capture (which internally calls learning-promote-auto and promotes)
   - Act: Call learning-promote-auto again (simulating the batch sweep in continue-finalize or seal)
   - Assert: Returns promoted === false with reason === "already_promoted", QUEEN.md still has content once

**QUEEN-03 Tests (wisdom in colony-prime prompt_section):**

7. "colony-prime includes promoted wisdom in prompt_section"
   - Setup: Create colony with pheromones.json, observe pattern 2x, promote it via learning-promote-auto
   - Act: Call colony-prime
   - Assert: result.ok === true, result.prompt_section contains the promoted content, result.prompt_section contains "QUEEN WISDOM" or "Eternal Guidance"

8. "colony-prime omits wisdom section when QUEEN.md has no promoted entries"
   - Setup: Create colony with pheromones.json but leave QUEEN.md empty (template only, no promoted content)
   - Act: Call colony-prime
   - Assert: result.ok === true, result.prompt_section does NOT contain "QUEEN WISDOM" (or the section is omitted/empty)

For the setupTestColony helper, extend with the same options pattern as pheromone-auto-emission.test.js. Include COLONY_STATE.json with a goal, session_id, current_phase, and minimal plan.phases array. Include pheromones.json with empty signals and instincts.
