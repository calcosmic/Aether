## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input -> classification | Worker status and error messages from CLI flags are untrusted input that enters classification function |
| Recovery log file -> CLI output | Per-phase JSON files are read and displayed; corrupted files should not crash the CLI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-01 | Tampering | classifyWorkerFailure input | mitigate | Function normalizes input and never panics; unknown statuses return safe default (RequiresAttempt) |
| T-94-02 | Tampering | Recovery log error messages | accept | Error messages come from worker output (not user input); SanitizeSignalContent already strips injection patterns |
| T-94-03 | Tampering | Recovery log file path | accept | Phase number is int, not string -- path traversal via recovery-log-{N}.json is not possible |
| T-94-04 | Information disclosure | recovery-log-read output | accept | Recovery log contains worker error messages -- no secrets, only colony operational data |
</threat_model>
