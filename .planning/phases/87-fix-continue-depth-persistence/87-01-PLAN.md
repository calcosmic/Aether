<objective>
Wire persisted verification depth into /ant-continue flow so it honors the depth selected at plan time.

Purpose: DEPTH-05 requires that the verification depth selected during /ant-plan is stored and honored by /ant-continue without the user re-specifying it. Currently, continue passes an empty string to resolveVerificationDepth at 4 call sites, causing it to fall through to the smart default instead of the user's plan-time selection.

Output: All continue flows (plan-only, full, blocked-recovery, and colony-prime context) read state.VerificationDepth and pass it to resolveVerificationDepth. The --verification-depth CLI flag on continueCmd is wired so users can override at continue time. Tests verify the behavior.
</objective>
