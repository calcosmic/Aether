  **Environment setup:**
  - Create a temp directory with `mktemp -d`
  - Override `HOME` to this temp dir (isolates from user's real ~/.aether/)
  - Set up cleanup via `trap ... EXIT`
  - Source `e2e-helpers.sh` for `extract_json` and assertion helpers
  - Use file-based result tracking (same pattern as test-lifecycle.sh `record_lifecycle_result`)

  **Step 1: Hub installation (simulates `npm install -g aether-colony`)**
  - Run `node bin/cli.js install --quiet` from the project root
  - Verify `$HOME/.aether/system/aether-utils.sh` exists
  - Verify `$HOME/.aether/system/` contains utils/ directory
  - Record PASS/FAIL

  **Step 2: Lay-eggs simulation (simulates `/ant:lay-eggs` on a fresh repo)**
  - Create a temp repo directory inside the test HOME: `mkdir -p "$TEST_REPO" && cd "$TEST_REPO" && git init -q`
  - Create `.aether/` in the test repo by copying from hub (this is what lay-eggs does):
    - `cp "$HOME/.aether/system/aether-utils.sh" "$TEST_REPO/.aether/"`
    - `cp -r "$HOME/.aether/system/utils/" "$TEST_REPO/.aether/utils/"` (if exists)
    - `mkdir -p "$TEST_REPO/.aether/data"`
    - `mkdir -p "$TEST_REPO/.aether/exchange"`
    - Copy exchange/ from system if exists
  - Run `queen-init` subcommand: `bash "$TEST_REPO/.aether/aether-utils.sh" queen-init 2>&1`
  - Verify QUEEN.md was created in the repo
  - Verify QUEEN.md contains template placeholder text (e.g., "No philosophies recorded yet"), NOT promoted entries
  - Record PASS/FAIL

  **Step 3: Colony init (simulates `/ant:init "smoke test"`)**
  - Write COLONY_STATE.json from the colony-state template (copy from hub templates if available, else write minimal fixture like test-lifecycle.sh does) with goal "fresh-install-smoke-test"
  - Run `session-init "" "fresh-install-smoke-test"` subcommand
  - Verify returns `ok:true`
  - Verify `session.json` created with `colony_goal` and `current_phase`
  - Record PASS/FAIL

  **Step 4: Signal flow (simulates pheromone operations during plan/build)**
  - Initialize pheromones from template: create empty `pheromones.json` with `{"signals":[],"midden":[]}`
  - Run `pheromone-write FOCUS "fresh install quality" --strength 0.8`
  - Verify returns `ok:true`
  - Run `pheromone-read`
  - Verify returns `ok:true` with at least 1 signal
  - Record PASS/FAIL

  **Step 5: Build simulation (simulates `/ant:build`)**
  - Seed `pheromones.json` with the FOCUS signal written in Step 4 (it should already be there from pheromone-write)
  - Run `pheromone-prime builder` to verify signal injection
  - Verify returns `ok:true`
  - Run `session-update "/ant:plan" "/ant:build 1" "Plan created"`
  - Verify returns `ok:true`
  - Run `session-update "/ant:build" "/ant:continue" "Build in progress"`
  - Verify returns `ok:true`
  - Record PASS/FAIL

  **Step 6: Continue simulation (simulates `/ant:continue`)**
  - Run `session-update "/ant:continue" "/ant:seal" "Phase completed"`
  - Verify returns `ok:true`
  - Run `milestone-detect`
  - Verify returns `ok:true`
  - Record PASS/FAIL

  - Write to external results file if --results-file provided
  - Exit 0 if all pass, exit 1 if any fail

  - It simulates lay-eggs by copying from the hub (as lay-eggs.md does)
