Create cmd/review_ledger_persistence_test.go with 3 test functions:

1. TestReviewLedgerPersistence:
   - Create a temp store (use the same pattern as review_ledger_test.go)
   - Write findings to 3 different domains (e.g., security, quality, testing)
   - Read back each domain's ledger
   - Verify: all written findings are present in the read output
   - Verify: findings have all expected fields (phase, finding_id, severity, description, timestamp)
   - Verify: domain count matches golden snapshot (7 domains)
   - Verify: max_findings_per_write matches golden (50)

2. TestReviewLedgerCrossPhaseAccumulation:
   - Create a temp store
   - Simulate "Phase 100" writing 2 findings to the security domain
   - Verify: security ledger has 2 entries
   - Simulate "Phase 101" writing 2 more findings to the security domain
   - Verify: security ledger now has 4 entries (accumulated, not overwritten)
   - Simulate "Phase 102" writing findings to a different domain (quality)
   - Verify: security still has 4 entries, quality has its entries
   - Verify: total unique domains with entries >= 2

3. TestReviewLedgerSessionResetSurvival:
   - Create a temp directory for store files
   - Create store instance A, write findings to 2 domains
   - Close/discard store instance A (simulate session end)
   - Create fresh store instance B pointing to the SAME directory
   - Read ledgers via store instance B
   - Verify: all findings written by instance A are readable by instance B
   - Verify: no data loss occurred across the "session reset"

All tests must use t.TempDir() for store isolation.
