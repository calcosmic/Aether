Create cmd/release_pipeline_test.go with 3 test functions:

1. TestReleasePipelineE2E:
   - Create a mock source checkout (temp dir with go.mod, cmd/aether/main.go, .aether/commands/, .claude/commands/ant/, .opencode/commands/ant/)
   - Create a mock hub directory (temp dir simulating ~/.aether/)
   - Create a mock binary file in the source checkout
   - Run the publish sync logic (call the internal sync functions directly, NOT subprocess)
   - Verify: all companion files from installSyncPairs exist in the mock hub
   - Verify: version file in hub matches source version
   - Run update --force logic from mock hub back to a fresh mock repo
   - Verify: no stale files remain (files in repo that are not in sync pairs should be removed)
   - Run install logic to a fresh mock home
   - Verify: all files present in the fresh home

2. TestPublishHubSync:
   - Narrower test focusing only on publish→hub sync
   - Create mock source checkout
   - Run publish sync
   - Verify each installSyncPair has its destination file/directory present in hub
   - Verify version agreement: the version written to hub matches the source version
   - Load golden snapshot and verify sync_pair_count matches actual

3. TestUpdateStaleFileCleanup:
   - Create a mock repo with companion files
   - Add an extra "stale" file that is NOT in any sync pair
   - Run update --force logic
   - Verify: the stale file is removed
   - Verify: all legitimate sync-pair files remain
