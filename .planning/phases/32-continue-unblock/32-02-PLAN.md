<objective>
Add stale report cleanup and verify end-to-end pipeline recovery so colonies can advance after abandoned builds are re-dispatched.

Purpose: When continue runs multiple times on a stuck phase, previous verification.json, gates.json, and continue.json files accumulate. These stale reports do not affect verification (the watcher reads the manifest, not reports), but they confuse users reviewing the phase directory. Clearing them before re-running ensures a clean state. The integration test proves the full recovery path works.

Output: cleanupStaleContinueReports function + 2 new tests (cleanup test + integration test).
</objective>
