<objective>
Add abandoned-build detection to the continue pipeline so it can distinguish "build never completed" from "verification failed" and produce actionable recovery messages.

Purpose: When a colony's build is abandoned mid-execution (all dispatches stuck at "spawned", stale BuildStartedAt), continue currently returns a generic "blocked" result. This plan adds a dedicated detection path that returns an "abandoned" result with clear recovery commands, without opening any bypass paths.

Output: `detectAbandonedBuild` function + abandoned result branch in `runCodexContinue` + 2 new tests.
</objective>

  <behavior>
    - When all manifest dispatches have status "spawned" and BuildStartedAt is older than 10 minutes, detectAbandonedBuild returns (true, duration, summary).

    - When BuildStartedAt is nil, detectAbandonedBuild returns (false, 0, "").
    - When elapsed time is under 10 minutes, detectAbandonedBuild returns (false, 0, "") -- build may still be running.
    - When abandoned is detected, runCodexContinue returns a result map with "abandoned": true, "blocked": true, "advanced": false, and a "recovery" map with redispatch and reconcile commands.
    - When abandoned is detected, runStatus is set to "blocked-abandoned" for the spawn run record.
    - The abandoned result includes "stale_duration" as a human-readable string.
    - The abandoned result includes "task_ids" listing all task IDs from the manifest dispatches that need re-dispatch.
  </behavior>
