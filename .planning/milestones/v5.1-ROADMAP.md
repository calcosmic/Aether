# Milestone v5.1: System Simplification

**Status:** SHIPPED 2026-02-06
**Phases:** 33-40
**Total Plans:** 22

## Overview

Transform Aether from a 7,400-line over-engineered system to a lean 1,800-line implementation based on the M4L-AnalogWave postmortem findings. The postmortem identified that 70% of context was consumed by framework overhead, state fell out of sync at context boundaries, and mathematical models (Bayesian, exponential decay) added complexity without value.

## Milestone Summary

**Key Decisions:**
- Postmortem-driven simplification â€” reduce from 7,400 to 1,800 lines
- Follow postmortem implementation order (state first, then commands)
- Preserve nested structure (plan, memory, errors) for semantic clarity
- Events as pipe-delimited strings per SIMP-01 requirement
- Build writes only EXECUTING state before workers
- SUMMARY.md existence as primary completion signal (passive detection)
- Signal keywords instead of sensitivity tables
- phase_end as default signal expiration (not wall-clock based)
- Priority levels (high/normal/low) replace numeric strength
- Filter expired signals on read (no cleanup command needed)
- Single-pass surface scan replaces multi-colonizer spawning

**Issues Resolved:**
- State fell out of sync at context boundaries (fixed with start-of-next-command pattern)
- 70% context consumed by framework overhead (reduced to ~20%)
- Mathematical models added complexity without value (removed Bayesian, exponential decay)

- None

- None (stale directory cleaned up)
