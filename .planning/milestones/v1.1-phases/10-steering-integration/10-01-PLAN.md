**oracle.sh -- Add read_steering_signals function (place after compute_trust_scores, before check_convergence):**

Create `read_steering_signals()` that takes `$1` as aether_root:
1. Check if `$aether_root/.aether/aether-utils.sh` exists, return empty string if not (graceful degradation)
2. Call `bash "$aether_root/.aether/aether-utils.sh" pheromone-read 2>/dev/null || echo '{"signals":[]}'` to get active signals
3. Extract signal array from the JSON response using `jq -c '.result.signals // .signals // []'` (handles both json_ok wrapper and raw formats)
4. Count signals. If 0, return empty string
5. Format REDIRECT signals (max 2, sorted by effective_strength desc): `- [XX%] "signal text"`
6. Format FOCUS signals (max 3, sorted by effective_strength desc): same format
7. Format FEEDBACK signals (max 2, sorted by effective_strength desc): same format
8. Wrap in a markdown directive block starting with `## Active Steering Signals` followed by each non-empty signal type with headers (`**REDIRECT (Hard constraints -- MUST follow):**`, `**FOCUS (Prioritize these areas):**`, `**FEEDBACK (Adjust approach):**`)
9. End with instruction text: "When selecting your target question, PRIORITIZE questions related to FOCUS areas. REDIRECT signals are hard constraints -- adjust your approach to comply. FEEDBACK signals are suggestions -- incorporate where appropriate."
10. End with `---` separator

Use the exact jq patterns from the research document code examples section for signal formatting. Use `|| echo ""` for graceful jq failures. Use the `$'\n'` bash pattern for newlines in string concatenation.

**oracle.md -- Add steering response section:**

After the "## Phase Directive" section and before "## Instructions", add a new section:

```markdown
## Steering Signals

If steering signals appear above this prompt, they were emitted by the user:

- **REDIRECT** signals are HARD CONSTRAINTS. You MUST follow them. If a REDIRECT
  conflicts with your planned approach, change your approach.
- **FOCUS** signals indicate priority areas. When choosing your target question,
  prefer questions related to focus areas. If no questions match, fall back to
  default targeting (lowest-confidence).

  into your research approach and output style.

```
