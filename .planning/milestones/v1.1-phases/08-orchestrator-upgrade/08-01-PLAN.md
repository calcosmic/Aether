  <files>.aether/oracle/oracle.sh</files>
  <action>
Add 8 new functions to oracle.sh, placed between the existing `build_oracle_prompt` function and the state.json existence check (line ~202). Then restructure the main loop to use them.

**New functions to add (in order):**

1. `compute_convergence()` -- Takes plan_file and state_file as args. Computes three structural metrics from plan.json using jq:
   - Gap resolution rate: `(answered + partial_with_confidence>=70) * 100 / total_questions`. Use integer arithmetic. Handle zero-questions edge case (return 100).
   - Coverage completeness: `questions_with_nonempty_iterations_touched * 100 / total_questions`. Handle zero-questions edge case.
   - Novelty delta: `current_total_findings - prev_findings_count_from_state.json.convergence`. Read prev_findings_count from state.json `.convergence.prev_findings_count // 0`.
   - Output a JSON object via `jq -n` with `gap_resolution_pct`, `coverage_pct`, `novelty_delta`, `total_findings`.

2. `update_convergence_metrics()` -- Takes state_file and plan_file as args. Calls `compute_convergence`, then updates state.json:
   - Create/update `.convergence` object with: `prev_findings_count` (set to current total findings), `prev_overall_confidence` (set to current `.overall_confidence`).
   - Append to `.convergence.history` array: `{iteration, novelty_delta, confidence_delta (current - prev overall_confidence), gap_resolution_pct, coverage_pct, phase}`. Read current iteration from state.json.
   - Compute composite score: `gap_resolution * 0.4 + coverage * 0.3 + (if novelty_delta <= 1 then 100 else 0) * 0.3` -- note: the novelty component rewards LOW novelty (research has stopped finding new things at high coverage). All done in integer arithmetic scaled by 100 (so multiply by 40/30/30 and divide by 100).
   - Store composite score in `.convergence.composite_score` and set `.convergence.converged` to true/false.
   - Use jq `--argjson` for all numeric values. Write via temp file + mv pattern (consistent with existing oracle.sh style).

3. `check_convergence()` -- Takes state_file as arg. Returns 0 (true) if: composite_score >= 85 AND at least 2 of the last entries in convergence.history have novelty_delta <= 1. Returns 1 (false) otherwise. Check convergence.history length >= 2 before accessing last-2 entries.
   - Support ORACLE_CONVERGENCE_THRESHOLD env var override (default 85).

4. `detect_diminishing_returns()` -- Takes state_file as arg. Reads last 3 entries from `.convergence.history`. Counts entries with novelty_delta at or below a phase-adjusted threshold:
   - `investigate` phase: threshold is 0 (any new finding counts as progress)
   - `survey/synthesize/verify` phases: threshold is 1
   - If 3 consecutive entries meet the threshold AND history has >= 3 entries:
     - If current phase is `survey` or `investigate`: echo "strategy_change"
     - If current phase is `synthesize` or `verify`: echo "synthesize_now"
   - Otherwise: echo "continue"
   - Support ORACLE_DR_WINDOW env var override for the window size (default 3).

5. `validate_and_recover()` -- Takes file path as arg. Check if file is valid JSON (`jq -e . "$file" >/dev/null 2>&1`). If valid, return 0. If invalid:
   - Try `${file}.pre-iteration` backup -- verify it exists and is valid JSON, then cp it over the file.
   - If pre-iteration backup fails, try `source "$AETHER_ROOT/.aether/utils/atomic-write.sh"` and call `restore_backup "$file"`.
   - Log all recovery actions (echo "WARNING: ..." to stderr with `>&2`).
   - Return 0 on successful recovery, 1 if unrecoverable.

6. `build_synthesis_prompt()` -- Takes reason as arg (one of: "converged", "stopped", "max_iterations", "interrupted", "corruption"). Outputs the synthesis directive text followed by `cat "$SCRIPT_DIR/oracle.md"`. The directive tells the AI to rewrite synthesis.md as a structured final report with: Executive Summary, Findings by Question (with confidence), Open Questions, Methodology Notes. Also instructs the AI to update state.json status to "complete" (if converged) or "stopped" (otherwise). Include instruction: "If any state file is unreadable, skip it and work with what you have."

7. `run_synthesis_pass()` -- Takes reason as arg. Updates state.json status and stop_reason using jq before the AI call. Then invokes `build_synthesis_prompt "$reason" | timeout 180 $AI_CMD 2>&1 | tee /dev/stderr` (using `timeout` command with 180 second limit). If timeout exits non-zero, log warning but do not fail. Then call `generate_research_plan` one final time. If timeout command is not available (check with `command -v timeout`), skip the timeout wrapper and run AI_CMD directly.

8. `cleanup_and_synthesize()` -- The trap handler. Uses the INTERRUPTED flag pattern: if INTERRUPTED is already true, just `exit 130`. Otherwise set INTERRUPTED=true, echo "Oracle interrupted. Running synthesis pass...", call `run_synthesis_pass "interrupted"`, echo the result file paths, then `exit 130`.

**Main loop restructuring:**

Replace the existing main loop (lines ~257-319) with the upgraded version. Keep all existing behavior (stop file check, AI invocation, iteration increment, phase transition, research-plan regeneration, completion signal check, sleep 2) but add these insertions:

- Before the `for` loop: `INTERRUPTED=false` and `trap cleanup_and_synthesize SIGINT SIGTERM`
- After stop file check: call `run_synthesis_pass "stopped"` before the break, then `exit 0` instead of `break`
- Before AI invocation: pre-iteration backup (`cp "$STATE_FILE" "$STATE_FILE.pre-iteration"` and same for PLAN_FILE)
- After AI invocation, REPLACE the warning-only jq validation with: `if ! validate_and_recover "$STATE_FILE" || ! validate_and_recover "$PLAN_FILE"; then run_synthesis_pass "corruption"; exit 1; fi`
- After phase transition: call `update_convergence_metrics "$STATE_FILE" "$PLAN_FILE"`
- After convergence metrics: call diminishing returns detection, handle "strategy_change" (force phase to "synthesize" via jq) and "synthesize_now" (call run_synthesis_pass "converged" and exit 0)
- After diminishing returns: call `check_convergence "$STATE_FILE"`, if true call `run_synthesis_pass "converged"` and exit 0
- After completion signal check: wrap the existing `<oracle>COMPLETE</oracle>` check to also call run_synthesis_pass "converged" before exit
- After the loop (max iterations): replace `exit 1` with `run_synthesis_pass "max_iterations"` then `exit 0`

**Important constraints:**
- Keep `set -e` at top (existing)
- All jq writes use `> "$file.tmp" && mv "$file.tmp" "$file"` pattern (existing convention)
- Bash 3.2 compatible -- no associative arrays, no `[[ ]]` where `[ ]` suffices, no `declare -A`
- ORACLE_CONVERGENCE_THRESHOLD defaults to 85, ORACLE_DR_WINDOW defaults to 3 (env var overrides for tuning)
- The convergence object in state.json is optional -- oracle.sh creates it on first use with `// {}` and `// []` jq fallbacks
