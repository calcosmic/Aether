<objective>
Add research strategy templates and adaptive output structure to the oracle. The wizard gains a template selection question (tech-eval, architecture-review, bug-investigation, best-practices, or custom). The selected template writes to state.json and drives both initial question decomposition (plan.json pre-population) and final synthesis output structure (template-specific sections in build_synthesis_prompt). All templates add confidence grouping to findings.

Purpose: Different research types produce different output structures -- a tech evaluation should produce a comparison matrix and recommendation, not the same generic report as a bug investigation.
Output: Template wizard question, template-aware build_synthesis_prompt, template-derived default questions, confidence grouping in all synthesis output.
</objective>

<task type="auto">
  <name>Task 1: Add template-aware build_synthesis_prompt to oracle.sh</name>
  <files>.aether/oracle/oracle.sh</files>
  <action>
Update the existing `build_synthesis_prompt` function in oracle.sh to read the template field from state.json and emit template-specific output section requirements. Follow Pattern 3 from 11-RESEARCH.md.

1. **Read template from state.json** at the top of build_synthesis_prompt:
   ```bash
   local template
   template=$(jq -r '.template // "custom"' "$STATE_FILE" 2>/dev/null || echo "custom")
   ```

2. **Replace the existing hardcoded section requirements** with a case statement on `$template`. Keep the existing synthesis preamble (SYNTHESIS PASS header, reason, file list, REWRITE instruction). After the preamble, emit template-specific sections:

   - `tech-eval`: Executive Summary, Comparison Matrix, Pros and Cons, Adoption Assessment, Migration/Integration Path, Recommendation, Open Questions, Sources
   - `architecture-review`: Executive Summary, Component Map, Dependency Analysis, Risk Assessment, Scalability Analysis, Improvement Recommendations, Open Questions, Sources
   - `bug-investigation`: Executive Summary, Reproduction Steps, Root Cause Analysis, Impact Assessment, Fix Recommendations, Related Issues, Open Questions, Sources
   - `best-practices`: Executive Summary, Best Practice Benchmark, Current State Assessment, Gap Analysis, Action Plan, Open Questions, Sources
   - `custom` or `*` (default): Preserve the EXACT current section requirements (Executive Summary, Findings by Question with confidence %, Open Questions, Methodology Notes, Sources). This is the fallback -- existing behavior must not change.

   Use the exact section text from 11-RESEARCH.md Pattern 3 -- it has the precise section descriptions for each template.

3. **Add common confidence grouping directive** after the template-specific sections (applies to ALL templates including custom):
   ```
   ### Confidence Grouping:
   Within each findings section, group findings by confidence level:
   - **High confidence (80%+)** -- list first with full citations
   - **Medium confidence (50-79%)** -- list with caveats noted
   - **Low confidence (<50%)** -- list as tentative/unverified

   Use inline citations [S1], [S2] linking findings to their sources.
   Flag single-source findings with (single source) marker.
   ```

4. **Keep the existing state.json status update instruction and oracle.md cat** after the common directives (already present at the end of build_synthesis_prompt).

Important: The custom/default case must produce byte-identical output to the current build_synthesis_prompt (minus the new confidence grouping addition, which is additive for all templates). Do NOT change the existing generic section text -- copy it exactly as is for the custom branch.
  </action>
