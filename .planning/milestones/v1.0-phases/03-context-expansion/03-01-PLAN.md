<task type="auto">
  <name>Task 1: Add CONTEXT.md decision extraction to colony-prime</name>
  <files>.aether/aether-utils.sh</files>
  <action>
Add a CONTEXT.md decision extraction block to the colony-prime subcommand in aether-utils.sh, inserting it AFTER the phase learnings block (ends around line 7692) and BEFORE the pheromone signals section (starts around line 7694).

The block must:
1. Set `cp_ctx_file="$AETHER_ROOT/.aether/CONTEXT.md"`
2. Check `[[ -f "$cp_ctx_file" ]]` before any parsing (skip silently if missing)
3. Use awk to extract rows from the "## Recent Decisions" markdown table:
   - Match rows starting with `| YYYY-MM-DD` (date-prefixed)
   - Skip the table header row (`| Date |`) and separator row (`|------|`)
   - Stop at the next `---` section divider
   - Extract Decision (field 3) and Rationale (field 4) columns
   - Format each as: `decision (rationale)` or just `decision` if rationale is empty/"-"
4. Cap decisions: 5 in non-compact mode, 3 in compact mode (use `cp_compact` variable already available)
   - Take the LAST N rows (most recent decisions are at the bottom of the table)
   - Use `tail -n $cp_max_decisions` on the extracted output
5. Format the section with markers:
   ```
   --- KEY DECISIONS (Active Decisions) ---
   - decision text (rationale)
   - another decision
   --- END KEY DECISIONS ---
   ```
6. Only append to `cp_final_prompt` if at least one decision was extracted (no empty section headers)
7. Count decisions into `cp_decision_count` and append to `cp_log_line`: `cp_log_line="$cp_log_line, $cp_decision_count decisions"`

Reference the code examples in 03-RESEARCH.md for exact awk and formatting patterns. Follow the same conditional-section pattern established in Phase 2 for learnings: check count > 0, then append section with markers.

Comment the block with: `# === CONTEXT.md decision injection (CTX-01) ===` and `# === END CONTEXT.md decision injection ===`
  </action>
  <verify>
Run `bash .aether/aether-utils.sh colony-prime 2>/dev/null | jq -r '.result.prompt_section'` from a colony with CONTEXT.md containing decisions. Output should include "KEY DECISIONS" section with decision text. Run from a colony WITHOUT CONTEXT.md -- no error, no "KEY DECISIONS" in output.
  </verify>
  <done>colony-prime prompt_section includes KEY DECISIONS section with extracted decision text when CONTEXT.md has decisions, omits it when file is missing or table is empty, and log_line includes decision count</done>
</task>

<task type="auto">
  <name>Task 2: Add blocker flag injection to colony-prime</name>
  <files>.aether/aether-utils.sh</files>
  <action>
Add a blocker flag injection block to colony-prime, inserting it AFTER the CONTEXT.md decisions block (from Task 1) and BEFORE the pheromone signals section.

The block must:
1. Set `cp_flags_file="$DATA_DIR/flags.json"`
2. Check `[[ -f "$cp_flags_file" ]]` before any parsing (skip silently if missing)
3. Get the current phase number from `cp_current_phase` (already available in colony-prime scope)
4. Use jq to filter flags.json:
   - Select entries where `type == "blocker"` AND `resolved_at == null` AND (`phase == $current_phase` OR `phase == null`)
   - Format each as: `[source: {source}] {title}\n  {description}`
   - Use `--argjson phase "$cp_current_phase"` for phase comparison
5. Cap blockers: 3 in non-compact mode, 2 in compact mode
6. Format with DISTINCT section markers (NOT "REDIRECT" -- must be distinguishable per success criteria #4):
   ```
   --- BLOCKER WARNINGS (Unresolved Build Blockers) ---
   These are critical issues that MUST be addressed. Treat as REDIRECT-priority.
   [source: verification] Title of the blocker
     Description of the blocker issue
   --- END BLOCKER WARNINGS ---
   ```
7. Only append to `cp_final_prompt` if at least one blocker was found (no empty section headers)
8. Count blockers into `cp_blocker_count` and append to `cp_log_line`: `cp_log_line="$cp_log_line, $cp_blocker_count blockers"`

The "BLOCKER WARNINGS" header and `[source: ...]` prefix format are deliberately different from the REDIRECT pheromone format (`REDIRECT (HARD CONSTRAINTS - MUST follow):` with `[strength]` prefix). This distinction is a requirement.

Comment the block with: `# === Blocker flag injection (CTX-02) ===` and `# === END blocker flag injection ===`
